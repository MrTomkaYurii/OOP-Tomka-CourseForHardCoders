## 3.4 — Структури

### Поточні проблеми
- Немає чіткої відповіді: "коли struct замість class"
- WorkSchedule є, але не пояснено чому саме struct
- `with` оператор показаний, але не пов'язаний з 3.5

### Нова структура 3.4

```
1. Питання: "чому WorkSchedule — struct, а не class?"
2. Порівняльна таблиця: struct vs class (семантика, копіювання, наслідування, пам'ять)
3. Синтаксис struct — WorkSchedule
4. Три способи ініціалізації: default, конструктор, with
5. C# 10+ parameterless constructor — нова можливість, окремий блок
6. with оператор → підводка до 3.5 (копіювання)
7. Правило: маленький + незмінний + value-semantic = struct
```
