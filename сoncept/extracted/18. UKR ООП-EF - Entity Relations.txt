Основною метою є навчити студентів налаштовувати взаємозв'язки між об'єктами даних, такі як один до одного, один до багатьох та багато до багатьох, використовуючи можливості, що надає Entity Framework.Програмне забезпечення: ОС Windows/Linux, Visual Studio, Visual Studio CodeХІД РОБОТИЗавдання 1.Система керування студентамиВаше завдання - створити базу даних для студентської системи, використовуючи підхід EF Core Code First. Вона повинна виглядати наступним чином:Ваші простори імен мають бути такими:- P01_StudentSystem - для вашого класу Startup, якщо він у вас є- P01_ StudentSystem.Data - для вашого DbContext- P01_ StudentSystem.Data.Models - для ваших моделейВащі моделі повинні виглядати наступним ичном:StudentSystemContext – ваш DbContextStudent:StudentIdName (up to 100 characters, unicode)PhoneNumber (exactly 10 characters, not unicode, not required)RegisteredOnBirthday (not required)Course:CourseIdName (up to 80 characters, unicode)Description (unicode, not required)StartDateEndDatePriceResource:ResourceIdName (up to 50 characters, unicode)Url (not unicode)ResourceType (enum – can be Video, Presentation, Document or Other)CourseIdHomework:HomeworkIdContent (string, linking to a file, not unicode)ContentType (enum – can be Application, Pdf or Zip)SubmissionTimeStudentIdCourseIdStudentCourse – проміжна таблиця між Students та CoursesВзаємозв’язки:One student пов’язаний з багатьма CourseEnrollmentsOne student пов’язаний з багатьма HomeworkSubmissionsOne course пов’язаний з багатьма StudentsEnrolledOne course пов’язаний з багатьма ResourcesOne course пов’язаний з багатьма HomeworkSubmissionsЗавдання 2. Football BettingYour task is to create a database for a Football Bookmaker System, using the Code First approach. It should look like this:Ваші простори імен мають бути такими:- P03_FootballBetting - для вашого класу Startup, якщо він у вас є- P03_ FootballBetting.Data - для вашого DbContext- P03_ FootballBetting.Data.Models - для ваших моделейВаші моделі повинні бути:Team – TeamId, Name, LogoUrl, Initials (JUV, LIV, ARS…), Budget, PrimaryKitColorId, SecondaryKitColorId, TownIdColor – Id, NameTown – Id, Name, CountryIdCountry – CountryId, NamePlayer – Id, Name, SquadNumber, TeamId, PositionId, IsInjuredPosition – PositionId, NamePlayerStatistic – GameId, PlayerId, ScoredGoals, Assists, MinutesPlayedGame – GameId, HomeTeamId, AwayTeamId, HomeTeamGoals, AwayTeamGoals, DateTime, HomeTeamBetRate, AwayTeamBetRate, DrawBetRate, Result)Bet – BetId, Amount, Prediction, DateTime, UserId, GameIdUser – UserId, Username, Password, Email, Name, BalanceВзаємозв’язки мыж таблицями:Team має один PrimaryKitColor та один SecondaryKitColor.Color має багато PrimaryKitTeams та багато SecondaryKitTeams.Team знаходиться в одному Town.Town може приймати декілька Teams.Game має одну HomeTeam та одну AwayTeam, і Team може мати багато HomeGames та багато AwayGames.Town може бути розташований в одній Country, і одна Country може мати багато Towns.Player може грати за одну Team, і одна Team може мати багато Players.Player може грати на одній Position, і одна Position може бути зайнята багатьма Players.Один Player може грати у багатьох Games, і в кожній Game бере участь багато Players (PlayerStatistics).Багато Bets можуть бути розміщені на одну Game, але кожна Bet може бути лише на одну Game.Кожна Bet на задану Game повинна мати Prediction результат.Bet може бути зроблена лише одним User, і один User може зробити багато Bets.Моделі, дані та клієнт слід розділити на різні шари (проекти).
