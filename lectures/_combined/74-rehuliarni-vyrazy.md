
## 11.5. Регулярні вирази

Класи `StringBuilder` та `String` надають достатню функціональність для роботи з рядками. Однак .NET пропонує ще один потужний інструмент - регулярні вирази. Регулярні вирази представляють ефективний і гнучкий метод обробки великих текстів, дозволяючи в той же час істотно зменшити обсяги коду в порівнянні з використанням стандартних операцій з рядками.

Основна функціональність регулярних виразів у .NET зосереджена у просторі імен `System.Text.RegularExpressions`. А центральним класом під час роботи з регулярними виразами є клас `Regex`. Наприклад, у нас є певний текст і нам треба знайти у ньому всі словоформи якогось слова. З класом `Regex` це зробити дуже просто:

```csharp
using System.Text.RegularExpressions;

string s = "Бик тупогуб, тупогубенький бичок, у бика губа біла була тупа";
Regex regex = new Regex(@"туп(\w*)");
MatchCollection matches = regex.Matches(s);

if (matches.Count > 0)
{
    foreach (Match match in matches)
    {
        Console.WriteLine(match.Value);
    }
}
else
{
    Console.WriteLine("Збігів не знайдено");
}
```

Тут ми знаходимо в рядку всі словоформи слова `"туп"`. Конструктор об'єкта `Regex` передає регулярний вираз для пошуку. Далі ми розберемо деякі елементи синтаксису регулярних виразів, а поки що достатньо знати, що вираз `туп(\w*)` позначає знайти всі слова, які мають корінь `"туп"` і після якого може стояти різна кількість символів. Вираз `\w` означає алфавітно-цифровий символ, а зірочка після виразу вказує на невизначену їхню кількість - їх може бути один, два, три або взагалі не бути.

Метод `Matches` класу `Regex` приймає рядок, до якого треба застосувати регулярні вирази, та повертає колекцію знайдених збігів.

Кожен елемент такої колекції представляє об'єкт `Match`. Його властивість `Value` повертає знайдений збіг.

І в даному випадку ми отримаємо наступний консольний вивід

```text
тупогуб

```

### Параметр RegexOptions

Клас `Regex` має низку конструкторів, що дозволяють виконати початкову ініціалізацію об'єкта. Дві версії конструкторів як один із параметрів приймають перерахування `RegexOptions`. Деякі з значень, що приймаються цим переліком:

- `Compiled`: при встановленні цього значення регулярний вираз компілюється у складання, що забезпечує швидше виконання
- `CultureInvariant`: під час встановлення цього значення ігноруватиметься регіональна різниця
- `IgnoreCase`: під час встановлення цього значення ігноруватиметься регістр
- `IgnorePatternWhitespace`: видаляє з рядка пробіли і дозволяє коментарі, що починаються зі знака `#`
- `Multiline`: вказує, що текст треба розглядати в багаторядковому режимі. При такому режимі символи `"^"` і `"$"` збігаються, відповідно, з початком та кінцем будь-якого рядка, а не з початком та кінцем усього тексту
- `RightToLeft`: приписує читати рядок праворуч наліво
- `Singleline`: у цьому режимі символ `"."` відповідає будь-якому символу, у тому числі послідовності `"\n"`, яка здійснює перехід на наступний рядок

Наприклад:

```csharp
Regex regex = new Regex(@"туп(\w*)", RegexOptions.IgnoreCase);
```

За потреби можна встановити кілька параметрів:

```csharp
Regex regex = new Regex(@"туп(\w*)", RegexOptions.Compiled | RegexOptions.IgnoreCase);
```

### Синтаксис регулярних виразів

Розглянемо коротко деякі елементи синтаксису регулярних виразів:

- `^`: відповідність повинна починатися на початку рядка (наприклад, вираз `@"^пр\w*"` відповідає слову `"привіт"` у рядку `"привіт світ"`)
- `$`: кінець рядка (наприклад, вираз `@"\w*іт$"` відповідає слову `"світ"` у рядку `"привіт світ"`, тому що частина `"іт"` знаходиться в самому кінці)
- `.`: знак точки визначає будь-який одиночний символ (наприклад, вираз `"м.р"` відповідає слову `"мир"` або `"мор"`)
- `*`: попередній символ повторюється 0 і більше разів
- `+`: попередній символ повторюється 1 і більше разів
- `?`: попередній символ повторюється 0 або 1 раз
- `\s`: відповідає будь-якому символу пробілу
- `\S`: відповідає будь-якому символу, що не є пробілом
- `\w`: відповідає будь-якому алфавітно-цифровому символу
- `\W`: відповідає будь-якому не алфавітно-цифровому символу
- `\d`: відповідає будь-якій десятковій цифрі
- `\D`: відповідає будь-якому символу, що не є десятковою цифрою

Це лише невелика частина елементів.

Тепер подивимося деякі приклади використання. Візьмемо перший приклад із скоромовкою `"Бик тупогуб, тупогубенький бичок, у бика губа біла була тупа"` і знайдемо в ній усі слова, де зустрічається корінь `"губ"`:

```csharp
string s = "Бик тупогуб, тупогубенький бичок, у бика губа біла була тупа";
Regex regex = new Regex(@"\w*губ\w*");
```

Так як вираз `\w*` відповідає будь-якій послідовності алфавітно-цифрових символів будь-якої довжини, то цей вираз знайде всі слова, що містять корінь `"губ"`.

Другий простенький приклад - знаходження телефонного номера у форматі `111-111-1111`:

```csharp
string s = "456-435-2318";
Regex regex = new Regex(@"\d{3}-\d{3}-\d{4}");
```

Якщо ми точно знаємо, скільки певних символів має бути, то ми можемо явно вказати їх кількість у фігурних дужках: `\d{3}` - тобто в цьому випадку три цифри.

Ми можемо не тільки задати пошук за певними типами символів - пробіли, цифри, а й задати конкретні символи, які повинні входити до регулярного виразу. Наприклад, перепишемо приклад із номером телефону і явно вкажемо, які символи там мають бути:

```csharp
string s = "456-435-2318";
Regex regex = new Regex("[0-9]{3}-[0-9]{3}-[0-9]{4}");
```

У квадратних дужках визначається діапазон символів, які повинні в цьому місці зустрічатися. У результаті цей і попередній шаблони телефонного номера будуть еквівалентні.

Також можна встановити діапазон для алфавітних символів: `Regex regex = new Regex("[a-v]{5}");` - цей вираз буде відповідати будь-якому поєднанню п'яти символів, в якому всі символи знаходяться в діапазоні від `a` до `v`.

Можна також вказати окремі значення: `Regex regex = new Regex(@"[2]*-[0-9]{3}-\d{4}");`. Цей вираз буде відповідати, наприклад, такому номеру телефону `"222-222-2222"` (бо перші числа двійки)

З допомогою операції `|` можна задати альтернативні символи, наприклад:

```csharp
Regex regex = new Regex(@"(2|3){3}-[0-9]{3}-\d{4}");
```

Тобто перші три цифри можуть містити лише двійки чи трійки. Такий шаблон буде відповідати, наприклад, рядкам `"222-222-2222"` та `"323-435-2318"`. А ось рядок `"235-435-2318"` вже не підпадає під шаблон, оскільки однією з перших трьох цифр є цифра `5`.

Отже, у нас такі символи, як `*`, `+` і ряд інших, використовуються як спеціальні символи. І виникає питання, а що робити, якщо нам треба знайти рядки, де міститься точка, зірочка чи якийсь інший спеціальний символ? У цьому випадку нам треба просто екранувати ці символи слешем:

```csharp
Regex regex = new Regex(@"(2|3){3}\.[0-9]{3}\.\d{4}");
// цьому виразу відповідатиме рядок "222.222.2222"
```

### Перевірка на відповідність рядка формату

Нерідко постає завдання перевірити коректність даних, введених користувачем. Це може бути перевірка електронної адреси, номера телефону. Клас `Regex` надає статичний метод `IsMatch`, який дозволяє перевірити вхідний рядок із шаблоном на відповідність:

```csharp
using System.Text.RegularExpressions;

string pattern = @"^(?("")(""[^""]+?""@)|(([0-9a-z]((\.(?!\.))|[-!#\$%&'\*\+/=\?\^`\{\}\|~\w])*)(?<=[0-9a-z])@))" +
                 @"(?(\[)(\[(\d{1,3}\.){3}\d{1,3}\])|(([0-9a-z][-\w]*[0-9a-z]*\.)+[a-z0-9]{2,17}))$";

var data = new string[]
{
    "tom@gmail.com",
    "+12345678999",
    "bob@yahoo.com",
    "+13435465566",
    "sam@yandex.ru",
    "+43743989393"
};

Console.WriteLine("Email List");
for (int i = 0; i < data.Length; i++)
{
    if (Regex.IsMatch(data[i], pattern, RegexOptions.IgnoreCase))
    {
        Console.WriteLine(data[i]);
    }
}
```

Змінна `pattern` визначає регулярний вираз для перевірки адреси електронної пошти. Цей вираз пропонує нам Microsoft на сторінках MSDN.

Далі в циклі ми проходимо масивом рядків і визначаємо, які рядки відповідають цьому шаблону, тобто представляють валідну адресу електронної пошти. Для перевірки відповідності рядка шаблону використовується метод `IsMatch`: `Regex.IsMatch(data[i], pattern, RegexOptions.IgnoreCase)`. Останній параметр показує, що регістр можна ігнорувати. І якщо рядок відповідає шаблону, метод повертає `true`.

### Заміна та метод Replace

Клас `Regex` має метод `Replace`, який дозволяє замінити рядок, що відповідає регулярному виразу, іншим рядком:

```csharp
string text = "Мама мила раму. ";
string pattern = @"\s+";
string target = " ";
Regex regex = new Regex(pattern);
string result = regex.Replace(text, target);
Console.WriteLine(result);
```

Дана версія методу `Replace` приймає два параметри: рядок із текстом, де треба виконати заміну, та сам рядок заміни. Так як як шаблон вибрано вираз `\s+` (тобто наявність одного і більше пробілів), метод `Replace` проходить по всьому тексту і замінює кілька пробілів, що йдуть, одинарними.

Інший приклад - на вхід подається номер телефону у довільному форматі, і ми хочемо залишити в ньому лише цифри:

```csharp
string phoneNumber = "+1(876)-234-12-98";
string pattern = @"\D";
string target = "";
Regex regex = new Regex(pattern);
string result = regex.Replace(phoneNumber, target);
Console.WriteLine(result); // 18762341298
```

В даному випадку шаблон `\D` представляє будь-який символ, який не є цифрою. Будь-який такий символ замінюється на порожній рядок `""`, тобто в результаті рядка `"+1(876)-234-12-98"` ми отримаємо рядок `"18762341298"`.
