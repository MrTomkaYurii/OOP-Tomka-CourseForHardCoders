## 3.4. Структури

Поряд із класами структури представляють ще один спосіб створення власних типів даних у C#. Більше того, багато примітивних типів, наприклад, `int`, `double` і т.д., по суті є структурами.

## Визначення структури

Для визначення структури застосовується ключове слово `struct`:

```csharp
struct ім'я_структури
{
    // Елементи структури
}
```

Після слова `struct` йде назва структури і далі у фігурних дужках розміщуються елементи структури - поля, методи тощо.

Наприклад, визначимо структуру, яка називатиметься `Person` і яка представлятиме людину:

```csharp
struct Person
{
}
```

Як і класи, структури можуть зберігати стан як поля (змінні) і визначати поведінку як методи. Наприклад, додамо в структуру `Person` пару полів та метод:

```csharp
struct Person
{
    public string name;
    public int age;

    public void Print()
    {
        Console.WriteLine($"Ім'я: {name} Вік: {age}");
    }
}
```

В даному випадку визначено дві змінні - `name` і `age` для зберігання відповідно імені та віку людини та метод `Print` для виведення інформації про людину на консоль.

І як і у випадку з класами, для звернення до функціональності структури - полів, методів та інших компонентів структури застосовується точкова нотація - після об'єкта структури ставиться крапка, а потім вказується компонент структури:

```text
об'єкт.поле_структури
об'єкт.метод_структури(параметри_методу)
```

## Створення об'єкта структури

## Ініціалізація за допомогою конструктора

Для використання структури її необхідно ініціалізувати. Для ініціалізації створення об'єктів структури, як і у випадку класів, застосовується виклик конструктора з оператором `new`. Навіть якщо в коді структури не визначено жодного конструктора, проте вона має як мінімум один конструктор - конструктор за замовчуванням, який генерується компілятором. Конструктор не приймає параметрів і створює об'єкт структури зі значеннями за замовчуванням.

```csharp
new назва_структури();
```

Наприклад, створимо об'єкт структури `Person` за допомогою конструктора за замовчуванням:

```csharp
Person tom = new Person(); // Виклик конструктора
// або так
// Person tom = new();

tom.name = "Tom"; // Змінюємо значення за замовчуванням у полі name

tom.Print(); // Ім'я: Tom Вік: 0

struct Person
{
    public string name;
    public int age;

    public void Print()
    {
        Console.WriteLine($"Ім'я: {name} Вік: {age}");
    }
}
```

У разі створюється об'єкт `tom`. Для його створення викликається конструктор за замовчуванням, який встановлює значення за замовчуванням для його полів. Для числових даних це значення `0`, тому поле `age` матиме значення `0`. Для рядків це значення `null`, що вказує на відсутність значення. Але далі, якщо поля доступні (а в даному випадку, оскільки вони мають модифікатор `public`, вони доступні), ми можемо змінити їх значення. Так, тут полю `name` надається рядок `"Tom"`. Відповідно при виконанні методу `Print()` ми отримаємо наступний консольний вивід:

```text
Ім'я: Tom Вік: 0
```

## Безпосередня ініціалізація полів

Якщо всі поля структури доступні (як у випадку з полями структури `Person`, які мають модифікатор `public`), структуру можна ініціалізувати без виклику конструктора. У цьому випадку необхідно надати значення всім полям структури перед отриманням значень полів і зверненням до методів структури. Наприклад:

```csharp
Person tom; // не викликаємо конструктор
// ініціалізація полів
tom.name = "Sam";
tom.age = 37;

tom.Print(); // Ім'я: Sam Вік: 37

struct Person
{
    public string name;
    public int age;

    public void Print()
    {
        Console.WriteLine($"Ім'я: {name} Вік: {age}");
    }
}
```

## Ініціалізація полів за замовчуванням

Починаючи з версії C# 10, ми можемо безпосередньо ініціалізувати поля структури при їх визначенні (до C# 10 це робити було не можна):

```csharp
Person tom = new Person();
tom.Print(); // Ім'я: Tom Вік: 1

struct Person
{
    // ініціалізація полів значеннями за умовчанням - доступна з C# 10
    public string name = "Tom";
    public int age = 1;

    public Person() { }

    public void Print() => Console.WriteLine($"Ім'я: {name} Вік: {age}");
}
```

Однак навіть у цьому випадку, незважаючи на значення за замовчуванням, необхідно явно визначити та викликати конструктор, якщо ми хочемо використовувати ці значення.

## Конструктори структури

Як і клас, структура може визначати конструктори. Наприклад, додамо до структури `Person` конструктор:

```csharp
Person tom = new();
Person bob = new("Bob");
Person sam = new("Sam", 25);

tom.Print(); //!!!! Ім'я:  Вік: 0
bob.Print(); // Ім'я: Bob Вік: 1
sam.Print(); // Ім'я: Sam Вік: 25

struct Person
{
    public string name;
    public int age;

    public Person(string name = "Tom", int age = 1)
    {
        this.name = name;
        this.age = age;
    }

    public void Print() => Console.WriteLine($"Ім'я: {name} Вік: {age}");
}
```

У разі у структурі `Person` визначено конструктор із двома параметрами, яким надані значення за умовчанням. Проте зверніть увагу на створення першого об'єкта структури:

```csharp
Person tom = new(); // як і раніше використовується конструктор без параметрів за замовчуванням
tom.Print(); //!!!! Ім'я:  Вік: 0
```

Тут як і раніше застосовується конструктор за умовчанням, тоді як для ініціалізації інших двох змінних структури застосовується явно визначений конструктор.

Однак, починаючи з версії C# 10, ми можемо визначити свій конструктор без параметрів:

```csharp
Person tom = new();

tom.Print(); // Ім'я: Tom Вік: 37

struct Person
{
    public string name;
    public int age;

    public Person()
    {
        name = "Tom";
        age = 37;
    }

    public void Print() => Console.WriteLine($"Ім'я: {name} Вік: {age}");
}
```

Варто зазначити, що до версії C# 11 щодо конструктора структури в ньому необхідно було ініціалізувати всі поля структури, починаючи з версії C# 11 це робити необов'язково.

Якщо нам потрібно викликати конструктори з різною кількістю параметрів, то ми можемо, як і у випадку з класами, викликати їх по ланцюжку:

```csharp
Person tom = new();
Person bob = new("Bob");
Person sam = new("Sam", 25);

tom.Print(); // Ім'я: Tom Вік: 1
bob.Print(); // Ім'я: Bob Вік: 1
sam.Print(); // Ім'я: Sam Вік: 25

struct Person
{
    public string name;
    public int age;

    public Person() : this("Tom")
    {
    }

    public Person(string name) : this(name, 1)
    {
    }

    public Person(string name, int age)
    {
        this.name = name;
        this.age = age;
    }

    public void Print() => Console.WriteLine($"Ім'я: {name} Вік: {age}");
}
```

Конструктори, як і раніше, повинні ініціалізувати значення всіх полів, однак оскільки при виклику будь-якого конструктора ланцюжок все одно закінчиться на останньому конструкторі, який виконує ініціалізацію, то ініціалізацію полів в інших конструкторах можна не робити. Консольний вивід програми:

```text
Ім'я: Tom Вік: 1
Ім'я: Bob Вік: 1
Ім'я: Sam Вік: 25
```

## Ініціалізатор структури

Так само, як і для класу, можна використовувати ініціалізатор для створення структури:

```csharp
Person tom = new Person { name = "Tom", age = 22 };

tom.Print(); // Ім'я: Tom Вік: 22

struct Person
{
    public string name;
    public int age;

    public void Print() => Console.WriteLine($"Ім'я: {name} Вік: {age}");
}
```

При використанні ініціалізації спочатку викликається конструктор без параметрів: якщо ми явно не визначили конструктор без параметрів, то викликається конструктор за замовчуванням. А потім його полям надаються відповідні значення.

## Копіювання структури за допомогою with

Якщо нам необхідно скопіювати в один об'єкт структури значення з іншого з невеликими змінами, ми можемо використовувати оператор `with`:

```csharp
Person tom = new Person { name = "Tom", age = 22 };
Person bob = tom with { name = "Bob" };
bob.Print(); // Ім'я: Bob Вік: 22
```

В даному випадку об'єкт `bob` отримує всі значення об'єкта `tom`, а потім після оператора `with` у фігурних дужках вказуються поля зі значеннями, які хочемо змінити.
