
## 14.5. Динамічне завантаження збірок та пізнє зв'язування

Під час створення програми для нього визначається набір збірок, які використовуватимуться. У проєкті вказуються посилання на ці збірки, і коли програма виконується, під час звернення до функціоналу цих збірок вони автоматично підвантажуються.

Але також ми можемо самі динамічно підвантажувати інші збірки, на які в проєкті немає посилань.

Для управління збірками у просторі імен `System.Reflection` є клас `Assembly`. З його допомогою можна завантажувати збірки, досліджувати їх.

Щоб динамічно завантажити збірку в програму, потрібно використовувати статичні методи `Assembly.LoadFrom()` або `Assembly.Load()`.

Метод `LoadFrom()` приймає як параметр шлях до збірки.

Припустимо, у нас є два проєкти:

![Два проєкти для прикладу динамічного завантаження збірки](/lecture-assets/docx/image106.png)

Нехай у проєкті `MyApp`, який компілюється у збірку `MyApp.dll`, є файл `Program.cs` з наступним кодом:

```csharp
Person tom = new Person("Tom");
Console.WriteLine($"Hello, {tom.Name}");

class Person
{
    public string Name { get; }
    public Person(string name) => Name = name;
}
```

В іншому проєкті досліджуємо збірку `MyApp.dll` на наявність у ній різних типів:

```csharp
using System.Reflection;

Assembly asm = Assembly.LoadFrom("MyApp.dll");

Console.WriteLine(asm.FullName);
// Отримуємо всі типи зі збірки MyApp.dll.
Type[] types = asm.GetTypes();
foreach (Type t in types)
{
    Console.WriteLine(t.Name);
}
```

У цьому випадку для дослідження вказується збірка `MyApp.dll`. Тут використано відносний шлях, так як збірка знаходиться в одній папці з додатком - у проєкті в каталозі `bin/Debug/net6.x`. Можна в принципі як ім'я вказати ім'я поточного додатку. У цьому випадку програма досліджуватиме саму себе. У будь-якому випадку варто враховувати, що завантаженню підлягають (принаймні в .NET 6.0) збірки з розширенням dll, але не exe.

І в моєму випадку я отримаю наступний консольний вивід:

```text
MyApp, Version=1.0.0.0, Culture=neutral, PublicKeyToken=null
EmbeddedAttribute
NullableAttribute
NullableContextAttribute
Program
Person
```

Як очевидно з виводу, повна назва збірки: `MyApp, Version=1.0.0.0, Culture=neutral, PublicKeyToken=null`. А сама збірка `MyApp.dll` містить п'ять типів - крім класу `Person` і класу `Program`, що неявно визначається, додається ще три автоматично генеровані класи.

Метод `Load()` діє аналогічно, тільки як його параметр передається дружнє ім'я збірки, яке нерідко збігається з ім'ям програми:

```csharp
Assembly asm = Assembly.Load("MyApp");
```

Отримавши всі типи збірки з допомогою методу `GetTypes()`, ми можемо застосувати до кожного типу всі ті методи, які були розглянуті в минулій темі.

### Пізнє зв'язування

За допомогою динамічного завантаження ми можемо реалізувати технологію пізнього зв'язування. Пізнє зв'язування дозволяє створювати екземпляри певного типу, а також використовувати його під час виконання програми.

Використання пізнього зв'язування є менш безпечним у тому плані, що при жорсткому кодуванні всіх типів (раннє зв'язування) на етапі компіляції ми можемо відстежити багато помилок. У той же час, пізнє зв'язування дозволяє створювати розширювані програми, коли додатковий функціонал програми невідомий, і його можуть розробити і підключити сторонні розробники.

Ключову роль пізньому зв'язуванні грає клас `System.Activator`. За допомогою статичного методу `Activator.CreateInstance()` можна створювати екземпляри заданого типу.

Наприклад, динамічно завантажимо збірку і викличемо у ній певний метод. Припустимо, завантажувана `MyApp.dll` представляє наступну програму:

```csharp
class Program
{
    static void Main(string[] args)
    {
        var number = 5;
        var result = Square(number);
        Console.WriteLine($"Квадрат {number} = {result}");
    }
    static int Square(int n) => n * n;
}
```

В даному випадку ми явно визначили клас `Program` з методом `Main`. І крім того, у класі `Program` визначено статичний метод `Square`, який як параметр приймає число та повертає його квадрат.

Тепер динамічно підключимо збірку з цією програмою в іншій програмі та викличемо її методи.

Нехай наша основна програма виглядатиме так:

```csharp
using System.Reflection;

Assembly asm = Assembly.LoadFrom("MyApp.dll");

Type? t = asm.GetType("Program");
if (t is not null)
{
    // Отримуємо метод Square.
    MethodInfo? square = t.GetMethod("Square", BindingFlags.NonPublic | BindingFlags.Static);

    // Викликаємо метод, передаємо йому значення для параметрів та отримуємо результат.
    object? result = square?.Invoke(null, new object[] { 7 });
    Console.WriteLine(result); // 49
}
```

Спочатку отримуємо посилання на досліджувану збірку у змінну `asm`:

```csharp
Assembly asm = Assembly.LoadFrom("MyApp.dll");
```

Потім за допомогою методу `GetType` отримуємо тип - клас `Program`, який знаходиться у збірці `MyApp.dll`:

```csharp
Type? t = asm.GetType("Program");
```

І насамкінець залишається викликати метод. По-перше, отримуємо сам метод:

```csharp
MethodInfo? square = t.GetMethod("Square", BindingFlags.NonPublic | BindingFlags.Static);
```

Оскільки метод `Square` приватний і статичний, то як другий параметр методу передаються прапори `BindingFlags.NonPublic | BindingFlags.Static`.

І потім за допомогою методу `Invoke` викликаємо його:

```csharp
object? result = square?.Invoke(null, new object[] { 7 });
```

Тут перший параметр представляє об'єкт, для якого викликається метод, а другий - набір параметрів як масиву `object[]`. Однак оскільки метод, що викликається, - статичний і не відноситься до якогось певного об'єкта, то першим аргументом у метод передається `null`.

Так як метод `Square` повертає деяке значення, ми можемо його отримати з методу у вигляді об'єкта типу `object`.

Якби метод не приймав параметрів, то замість масиву об'єктів використовувалося б значення `null`:

```csharp
method.Invoke(null, null);
```

У збірці `MyApp.dll` у класі `Program` також є й інший метод - метод `Main`, який також виконує деяку роботу. Викличемо тепер його:

```csharp
using System.Reflection;

Assembly asm = Assembly.LoadFrom("MyApp.dll");

Type? program = asm.GetType("Program");
if (program is not null)
{
    // Отримуємо метод Main.
    MethodInfo? main = program.GetMethod("Main", BindingFlags.NonPublic | BindingFlags.Static);

    // Викликаємо метод Main.
    main?.Invoke(null, new object[] { new string[] { } }); // Квадрат 5 = 25
}
```

Оскільки метод `Main` є статичним і непублічним, то до нього також застосовується бітова маска `BindingFlags.NonPublic | BindingFlags.Static`. І оскільки він як параметр приймає масив рядків, то при виклику методу передається відповідне значення:

```csharp
main.Invoke(null, new object[] { new string[] { } });
```
