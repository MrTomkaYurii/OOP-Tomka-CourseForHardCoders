# 3.2. Конструктори, ініціалізатори та деконструктори

## Створення конструкторів

Минулої статті для створення об'єкта використовувався конструктор за замовчуванням. Проте ми можемо визначити свої конструктори. Як правило, конструктор виконує ініціалізацію об'єкта. При цьому якщо в класі визначаються свої конструктори, він позбавляється конструктора за умовчанням.

На рівні коду конструктор представляє метод, який називається на ім'я класу, який може мати параметри, але для нього не треба визначати тип, що повертається. Наприклад, визначимо у класі `Person` найпростіший конструктор:

```csharp
Person tom = new Person(); // Створення об'єкта класу Person

tom.Print(); // Ім'я: Tom Вік: 37

class Person
{
    public string name;
    public int age;

    public Person()
    {
        Console.WriteLine("Створення об'єкта Person");
        name = "Tom";
        age = 37;
    }

    public void Print()
    {
        Console.WriteLine($"Ім'я: {name} Вік: {age}");
    }
}
```

Отже, тут визначено конструктор, який виводить на консоль деяке повідомлення та ініціалізує поля класу.

```csharp
public Person()
{
    Console.WriteLine("Створення об'єкта Person");
    name = "Tom";
    age = 37;
}
```

Конструктори можуть мати модифікатори, які зазначаються перед ім'ям конструктора. Так, у даному випадку, щоб конструктор був доступний поза класом `Person`, його визначено з модифікатором `public`.

Визначивши конструктор, ми можемо викликати його до створення об'єкта `Person`:

```csharp
Person tom = new Person(); // Створення об'єкта Person
```

В даному випадку вираз `Person()` якраз представляє виклик певного в класі конструктора (це більше не автоматичний конструктор за замовчуванням, якого клас тепер немає). Відповідно, при його виконанні на консолі буде виводитися рядок `"Створення об'єкта Person"`.

Подібним чином ми можемо визначати інші конструктори в класі. Наприклад, змінимо клас `Person` наступним чином:

```csharp
Person tom = new Person(); // Виклик першого конструктора без параметрів
Person bob = new Person("Bob"); // виклик 2-го конструктора з одним параметром
Person sam = new Person("Sam", 25); // Виклик 3-го конструктора з двома параметрами

tom.Print(); // Ім'я: Невідомо Вік: 18
bob.Print(); // Ім'я: Bob Вік: 18
sam.Print(); // Ім'я: Sam Вік: 25

class Person
{
    public string name;
    public int age;
    public Person() { name = "Невідомо"; age = 18; } // 1 конструктор
    public Person(string n) { name = n; age = 18; } // 2 конструктор
    public Person(string n, int a) { name = n; age = a; } // 3 конструктор

    public void Print()
    {
        Console.WriteLine($"Ім'я: {name} Вік: {age}");
    }
}
```

Тепер у класі визначено три конструктори, кожен із яких приймає різну кількість параметрів та встановлює значення полів класу. І ми можемо викликати один із цих конструкторів для створення об'єкта класу.

Консольний вивід цієї програми:

```text
Ім'я: Невідомо Вік: 18
Ім'я: Bob Вік: 18
Ім'я: Sam Вік: 25
```

Починаючи з версії C# 9 ми можемо скоротити виклик конструктора, прибравши з нього назву типу:

```csharp
Person tom = new(); // аналогічно new Person();
Person bob = new("Bob"); // аналогічно new Person("Bob");
Person sam = new("Sam", 25); // аналогічно new Person("Sam", 25);
```

Ключове слово `this`

Ключове слово `this` представляє посилання на поточний екземпляр/об'єкт класу. В яких ситуаціях воно нам може стати в нагоді?

```csharp
Person sam = new("Sam", 25);
sam.Print(); // Ім'я: Sam Вік: 25

class Person
{
    public string name;
    public int age;
    public Person() { name = "Невідомо"; age = 18; }
    public Person(string name) { this.name = name; age = 18; }
    public Person(string name, int age)
    {
        this.name = name;
        this.age = age;
    }
    public void Print() => Console.WriteLine($"Ім'я: {name} Вік: {age}");
}
```

У прикладі вище у другому та третьому конструкторі параметри називаються так само, як і поля класу. І щоб розмежувати параметри та поля класу, до полів класу звернення йде через ключове слово `this`. Так, у виразі

```csharp
this.name = name;
```

перша частина - `this.name` означає, що `name` - це поле поточного класу, а не назва параметра `name`. Якби у нас параметри та поля називалися по-різному, то використовувати слово `this` було б необов'язково. Також через ключове слово `this` можна звертатися до будь-якого поля або методу.

## Ланцюжок виклику конструкторів

У прикладі вище визначено три конструктори. Усі три конструктори виконують однотипні дії - встановлюють значення полів `name` та `age`. Але цих дій, що повторюються, могло бути більше. І ми можемо не дублювати функціональність конструкторів, а просто звертатися з одного конструктора до іншого через ключове слово `this`, передаючи потрібні значення для параметрів:

```csharp
class Person
{
    public string name;
    public int age;
    public Person() : this("Невідомо") // перший конструктор
    {}
    public Person(string name) : this(name, 18) // другий конструктор
    {}
    public Person(string name, int age) // Третій конструктор
    {
        this.name = name;
        this.age = age;
    }
    public void Print() => Console.WriteLine($"Ім'я: {name} Вік: {age}");
}
```

У разі перший конструктор викликає другий, а другий конструктор викликає третій. За кількістю та типом параметрів компілятор дізнається, який саме конструктор викликається. Наприклад, у другому конструкторі:

```csharp
public Person(string name) : this(name, 18)
{ }
```

йде звернення до третього конструктора, якому передаються два значення. Причому спочатку виконуватиметься саме третій конструктор, і лише потім код другого конструктора.

Варто відзначити, що у прикладі вище практично всі конструктори не визначають якихось інших дій, крім передають третьому конструктору деякі значення. Тому в реальності в даному випадку простіше залишити один конструктор, визначивши для параметрів значення за замовчуванням:

```csharp
Person tom = new();
Person bob = new("Bob");
Person sam = new("Sam", 25);

tom.Print(); // Ім'я: Невідомо Вік: 18
bob.Print(); // Ім'я: Bob Вік: 18
sam.Print(); // Ім'я: Sam Вік: 25

class Person
{
    public string name;
    public int age;
    public Person(string name = "Невідомо", int age = 18)
    {
        this.name = name;
        this.age = age;
    }
    public void Print() => Console.WriteLine($"Ім'я: {name} Вік: {age}");
}
```

І якщо при виклику конструктора ми не передаємо значення для якогось параметра, застосовується значення за замовчуванням.

## Ініціалізатори об'єктів

Для ініціалізації об'єктів класів можна використовувати ініціалізатори. Ініціалізатори представляють передачу у фігурних дужках значень доступним полям та властивостям об'єкта:

```csharp
Person tom = new Person { name = "Tom", age = 31 };
// або так
// Person tom = new() { name = "Tom", age = 31 };
tom.Print(); // Ім'я: Tom Вік: 31
```

За допомогою ініціалізації об'єктів можна надавати значення всім доступним полям і властивостям об'єкта в момент створення. При використанні ініціалізації слід враховувати наступні моменти:

- За допомогою ініціалізатора ми можемо встановити значення лише доступних з-поза класу полів та властивостей об'єкта. Наприклад, у прикладі вище поля `name` і `age` мають модифікатор доступу `public`, тому вони доступні з будь-якої частини програми.
- Ініціалізатор виконується після конструктора, тому якщо і в конструкторі, і в ініціалізаторі встановлюються значення тих самих полів і властивостей, то значення, що встановлюються в конструкторі, замінюються значеннями з ініціалізатора.

Ініціалізатори зручно застосовувати, коли поле або властивість класу представляє інший клас:

```csharp
Person tom = new Person { name = "Tom", company = { title = "Microsoft" } };
tom.Print(); // Ім'я: Tom Компанія: Microsoft

class Person
{
    public string name;
    public Company company;

    public Person()
    {
        name = "Undefined";
        company = new Company();
    }

    public void Print() => Console.WriteLine($"Ім'я: {name} Компанія: {company.title}");
}

class Company
{
    public string title = "Unknown";
}
```

Зверніть увагу, як встановлюється поле `company`:

```csharp
company = { title = "Microsoft" }
```

## Деконструктори

Деконструктори (не плутати з деструкторами) дозволяють виконати декомпозицію об'єкта на окремі частини.

Наприклад, нехай ми маємо наступний клас `Person`:

```csharp
class Person
{
    string name;
    int age;

    public Person(string name, int age)
    {
        this.name = name;
        this.age = age;
    }

    public void Deconstruct(out string personName, out int personAge)
    {
        personName = name;
        personAge = age;
    }
}
```

У цьому випадку ми могли б виконати декомпозицію об'єкта `Person` так:

```csharp
Person person = new Person("Tom", 33);

(string name, int age) = person;

Console.WriteLine(name);    // Tom
Console.WriteLine(age);     // 33
```

Значення змінним з деконструктора передаються за позицією. Тобто перше значення, що повертається у вигляді параметра `personName`, передається першій змінній - `name`, друге значення, що повертається, - змінній `age`.

Насправді деконструктори - це трохи більше, ніж синтаксичний цукор. Це все одно, що якби ми написали:

```csharp
Person person = new Person("Tom", 33);

string name;
int age;
person.Deconstruct(out name, out age);
```

При отриманні значень деконструктора нам необхідно надати стільки змінних, скільки деконструктор повертає значень. Однак буває, що не всі ці значення потрібні. І замість значень, що повертаються, ми можемо використовувати прочерк `_`. Наприклад, нам треба отримати лише вік користувача:

```csharp
Person person = new Person("Tom", 33);

(_, int age) = person;

Console.WriteLine(age);    // 33
```

Оскільки перше значення, що повертається, - це ім'я користувача, яке не потрібно, в цьому випадку замість змінної використовується прочерк.
