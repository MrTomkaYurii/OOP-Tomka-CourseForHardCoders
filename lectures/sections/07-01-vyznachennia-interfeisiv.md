## 7.1. Визначення інтерфейсів

Інтерфейс являє собою посилальний тип, який може визначити якийсь функціонал - набір методів і властивостей без реалізації. Потім цей функціонал реалізують класи і структури, які застосовують дані інтерфейси.

### Визначення інтерфейсу

Для визначення інтерфейсу використовується ключове слово interface. Як правило, назви інтерфейсів у C# починаються з головної букви I, наприклад, IComparable, IEnumerable (так звана угорська нотація), однак це не обов'язкова вимога, а більший стиль програмування.

Що може визначити інтерфейс? Загалом інтерфейси можуть визначати такі сутності:

- Методи
- Властивості
- Індексатори
- Події
- Статичні поля та константи (починаючи з версії C# 8.0)

Проте інтерфейси не можуть визначати нестатичні змінні. Наприклад, найпростіший інтерфейс, який визначає всі ці компоненти:

```csharp
interface IMovable
{
    // константа
    const int minSpeed = 0; // мінімальна швидкість
    // статична змінна
    static int maxSpeed = 60; // максимальна швидкість
    // метод
    void Move(); // рух
    // властивість
    string Name { get; set; } // назва
    delegate void MoveHandler(string message);
    // визначення делегату для події
    // подія
    event MoveHandler MoveEvent; // подія руху
}
```

У даному випадку визначено інтерфейс IMovable, який представляє деякий об'єкт, що рухається. Даний інтерфейс містить різні компоненти, які описують можливості об'єкта, що рухається. Тобто інтерфейс описує деякий функціонал, який повинен бути у об'єкта, що рухається.

Методи та властивості інтерфейсу можуть не мати реалізації, у цьому вони зближуються з абстрактними методами та властивостями абстрактних класів. В даному випадку інтерфейс визначає метод Move, який представлятиме деяке пересування. Він не має реалізації, не приймає жодних параметрів та нічого не повертає.

Те саме в цьому випадку стосується властивості Name. На перший погляд, вона схожа на автоматичну властивість. Але насправді це визначення властивості в інтерфейсі, яке не має реалізації, а не автовластивість.

### Модифікатори доступу

Ще один момент в оголошенні інтерфейсу: якщо його члени - методи та властивості не мають модифікаторів доступу, то фактично за умовчанням доступ public, оскільки мета інтерфейсу - визначення функціоналу для реалізації його класом. Це стосується також констант і статичних змінних, які в класах і структурах за замовчуванням мають модифікатор private. В інтерфейсах вони мають за замовчуванням модифікатор public. І наприклад, ми могли б звернутися до константи minSpeed та змінної maxSpeed інтерфейсу IMovable:

```csharp
Console.WriteLine(IMovable.maxSpeed); // 60
Console.WriteLine(IMovable.minSpeed); // 0
```

Крім того, починаючи з версії C# 8.0, ми можемо явно показувати модифікатори доступу до компонентів інтерфейсу:

```csharp
interface IMovable
{
    public const int minSpeed = 0; // мінімальна швидкість
    private static int maxSpeed = 60; // максимальна швидкість
    public void Move();
    protected internal string Name { get; set; } // назва
    public delegate void MoveHandler(string message); // визначення делегату для події
    public event MoveHandler MoveEvent; // подія руху
}
```

Як і класи, інтерфейси за замовчуванням мають рівень доступу internal, тобто такий інтерфейс доступний тільки в рамках поточного проекту. Але за допомогою модифікатора public ми можемо зробити інтерфейс загальнодоступним:

```csharp
public interface IMovable
{
    void Move();
}
```

### Реалізація за умовчанням

Також починаючи з версії C# 8.0 інтерфейси підтримують реалізацію методів і властивостей за замовчуванням. Це означає, що ми можемо визначити в інтерфейсах повноцінні методи та властивості, які мають реалізацію як в звичайних класах і структурах. Наприклад, визначив метод реалізації Move за замовчуванням:

```csharp
interface IMovable
{
    // реалізація методу по замовчуванню
    void Move()
    {
        Console.WriteLine("Walking");
    }
}
```

З реалізацією властивостей за замовчуванням в інтерфейсах справа складається кілька складніше, оскільки ми не можемо визначати в інтерфейсах нестатичні змінні, відповідно у властивостях інтерфейсу ми не можемо маніпулювати станом об'єкта. Тим не менш реалізацію за замовчуванням для властивості ми теж можемо визначити:

```csharp
interface IMovable
{
    // реалізація методу по замовчуванню
    void Move() => Console.WriteLine("Walking");
    // реалізація властивості по замовчуванню
    // властивість тільки для читання
    int MaxSpeed { get { return 0; } }
}
```

Варто відзначити, що якщо інтерфейс має приватні методи та властивості (що є з модифікатором private), то вони повинні мати реалізацію за замовчуванням. Це саме відноситься до статичних методів (не обов'язково приватним):

```csharp
Console.WriteLine(IMovable.MaxSpeed); // 60
IMovable.MaxSpeed = 65;
Console.WriteLine(IMovable.MaxSpeed); // 65
double time = IMovable.GetTime(500, 10);
Console.WriteLine(time); // 50
interface IMovable
{
    public const int minSpeed = 0; // мінімальна швидкість
    private static int maxSpeed = 60; // максимальна швидкість
    // знаходимо час, за який необхідно пройти відстань distance зі швидкістю speed
    static double GetTime(double distance, double speed) => distance / speed;
    static int MaxSpeed
    {
        get => maxSpeed;
        set
        {
            if (value > 0) maxSpeed = value;
        }
    }
}
```

### Додавання інтерфейсу

Слід зазначити, що в Visual Studio є спеціальний компонент для додавання нового інтерфейсу в окремому файлі. Для додавання інтерфейсу в проект можна натиснути праву кнопку миші в проекті і в контекстному меню, що з’явиться, вибрати Додати -> Новий елемент... і в діалоговому вікні додавання нового компонента вибрати пункт Інтерфейс:

![Додавання інтерфейсу у Visual Studio](_assets/_docx/image98.png)

Хоча ми також можемо додати стандартний файл класу або будь-який інший код файлу C# і не визначати інтерфейс.
