## 13.1. Відкладена ініціалізація та тип Lazy

Програма може використовувати безліч класів та об'єктів. Однак є ймовірність, що не всі об'єкти, що створюються, будуть використані. Особливо це стосується великих програм. Наприклад:

```csharp
class Reader
{
    Library library = new Library();

    public void ReadBook()
    {
        library.GetBook();
        Console.WriteLine("Читаємо паперову книгу");
    }

    public void ReadEbook()
    {
        Console.WriteLine("Читаємо книгу на комп'ютері");
    }
}

class Library
{
    private string[] books = new string[99];

    public void GetBook()
    {
        Console.WriteLine("Видаємо книгу читачеві");
    }
}
```

Є клас `Library`, який представляє бібліотеку та зберігає деякий набір книг у вигляді масиву. Є клас читача `Reader`, який зберігає посилання на об'єкт бібліотеки, де він записаний. У читача визначено два методи: для читання електронної книги та для читання звичайної книги. Для читання звичайної книги необхідно звернутися до методу класу `Library`, щоб отримати цю книгу.

Але якщо читачеві взагалі не доведеться читати звичайну книгу, а тільки електронні, наприклад, у наступному випадку:

```csharp
Reader reader = new Reader();
reader.ReadEbook();
```

У цьому випадку об'єкт `library` в класі читача ніяк не використовуватиметься і займатиме лише місце пам'яті. Хоча потреби в ньому не буде.

Для таких випадків у .NET визначено спеціальний клас `Lazy<T>`. Змінимо клас читача так:

```csharp
class Reader
{
    Lazy<Library> library = new Lazy<Library>();

    public void ReadBook()
    {
        library.Value.GetBook();
        Console.WriteLine("Читаємо паперову книгу");
    }

    public void ReadEbook()
    {
        Console.WriteLine("Читаємо книгу на комп'ютері");
    }
}
```

Клас `Library` залишається тим самим. Але тепер клас читача містить посилання на бібліотеку як об'єкт `Lazy<Library>`. А щоб звернутися до самої бібліотеки та її методів, треба використовувати вираз `library.Value` - це і є об'єкт `Library`.

Що змінює у поведінці класу `Reader` ця заміна? Розглянемо його застосування:

```csharp
Reader reader = new Reader();
reader.ReadEbook();
reader.ReadBook();
```

Безпосередньо об'єкт `Library` задіюється тут лише на третьому рядку у методі `reader.ReadBook()`, який викликає у свою чергу метод `library.Value.GetBook()`. Тому до третього рядка об'єкт `Library`, який використовується читачем, не буде створено. Якщо ми не будемо застосовувати в програмі метод `reader.ReadBook()`, то об'єкт бібліотеки тоді взагалі не буде створений, і ми уникнемо зайвих витрат пам'яті. Таким чином, `Lazy` гарантує нам, що об'єкт буде створений тільки тоді, коли в ньому є необхідність.
