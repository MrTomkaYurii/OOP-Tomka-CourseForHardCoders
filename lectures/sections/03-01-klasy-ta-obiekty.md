## 3.1. Класи та об'єкти

C# є повноцінною об'єктно-орієнтованою мовою. Це означає, що програму на C# можна у вигляді взаємозалежних взаємодіючих між собою об'єктів.

Описом об'єкта є клас, а об'єкт є екземпляр цього класу. Можна ще провести таку аналогію. У нас у всіх є деяке уявлення про людину, яка має ім'я, вік, якісь інші характеристики. Тобто, певний шаблон - цей шаблон можна назвати класом. Конкретне втілення цього шаблону може відрізнятися, наприклад, одні мають одне ім'я, інші - інше ім'я. І реально існуюча людина (фактично екземпляр цього класу) представлятиме об'єкт цього класу.

У принципі, раніше вже використовувалися класи. Наприклад, тип `string`, який представляє рядок, є фактично класом. Або, наприклад, клас `Console`, у якого метод `WriteLine()` виводить на консоль деяку інформацію. Тепер подивимося, як ми можемо визначати свої власні класи.

Насправді клас представляє новий тип, що визначається користувачем. Клас визначається за допомогою ключового слова `class`:

```csharp
class назва_класу
{
    // вміст класу
}
```

Після слова `class` йде ім'я класу і далі у фігурних дужках йде власне вміст класу. Наприклад, визначимо у файлі `Program.cs` клас `Person`, який представлятиме людину:

```csharp
class Person
{

}
```

![Рисунок з оригінального документа](assets/docx/image40.png)

Однак такий клас не є особливо показовим, тому додамо до нього деяку функціональність.

## Поля та методи класу

Клас може зберігати деякі дані. Для зберігання даних у класі застосовуються поля. Насправді поля класу - це змінні, визначені лише на рівні класу.

Крім того, клас може визначати деяку поведінку або дії, що виконуються. Для визначення поведінки у класі застосовуються методи.

Отже, додамо в клас `Person` поля та методи:

```csharp
class Person
{
    public string name = "Undefined"; // ім'я
    public int age;                    // вік

    public void Print()
    {
        Console.WriteLine($"Ім'я: {name}  Вік: {age}");
    }
}
```

В даному випадку в класі `Person` визначено поле `name`, яке зберігає ім'я, та поле `age`, яке зберігає вік людини. На відміну від змінних, визначених методами, поля класу можуть мати модифікатори, які вказуються перед полем. Так, у даному випадку, щоб усі поля були доступні поза класом `Person`, поля визначені модифікатором `public`.

При визначенні полів ми можемо надати їм деякі значення, як у прикладі вище у разі змінної `name`. Якщо поля класу не ініціалізовані, вони отримують значення за замовчуванням. Для змінних числових типів число 0.

Також у класі `Person` визначено метод `Print()`. Методи класу мають доступ до його поля, і в даному випадку звертаємося до полів класу `name` та `age` для виведення їх значення на консоль. І щоб цей метод був видно поза класом, він також визначений модифікатором `public`.

## Створення об'єкта класу

Після визначення класу ми можемо створювати об'єкти. Для створення об'єкта застосовуються конструктори. По суті, конструктори представляють спеціальні методи, які називаються так само як і клас, і які викликаються при створенні нового об'єкта класу і виконують ініціалізацію об'єкта. Загальний синтаксис виклику конструктора:

```csharp
new конструктор_класу(параметри_конструктору);
```

Спочатку йде оператор `new`, який виділяє пам'ять для об'єкта, а після нього йде виклик конструктора.

### Конструктор за замовчуванням

Якщо в класі не визначено жодного конструктора (як у випадку з нашим класом `Person`), для цього класу автоматично створюється порожній конструктор за замовчуванням, який не приймає жодних параметрів.

Тепер створимо об'єкт класу `Person`:

```csharp
Person tom = new Person(); // створення об'єкту класу Person

// визначення класу Person
class Person
{
    public string name = "Undefined";
    public int age;

    public void Print()
    {
        Console.WriteLine($"Ім'я: {name}  Вік: {age}");
    }
}
```

Для створення об'єкта `Person` використовується вираз `new Person()`. У результаті після виконання цього виразу в пам'яті буде виділено ділянку, де зберігатимуться всі дані об'єкта `Person`. А змінна `tom` отримає посилання на створений об'єкт, і через цю змінну ми можемо використовувати цей об'єкт і звертатися до його функціональності.

## Звернення до функціональності класу

Для звернення до функціональності класу - полів, методів (а також інших елементів класу) застосовується точкова нотація точки - після об'єкта класу ставиться крапка, а потім елемент класу:

```text
Об'єкт.поле_класу
Об'єкт.метод_класу(параметри методу)
```

Наприклад, звернімося до полів та методів об'єкта `Person`:

```csharp
Person tom = new Person(); // створення об'єкта класу Person

// Отримуємо значення полів у змінні
string personName = tom.name;
int personAge = tom.age;
Console.WriteLine($"Ім'я: {personName}  Вік: {personAge}");
// Ім'я: Undefined  Вік: 0

// встановлюємо нові значення полів
tom.name = "Tom";
tom.age = 37;

// звертаємося до методу Print
tom.Print(); // Ім'я: Tom  Вік: 37

class Person
{
    public string name = "Undefined";
    public int age;

    public void Print()
    {
        Console.WriteLine($"Ім'я: {name}  Вік: {age}");
    }
}
```

Консольний вивід цієї програми:

```text
Ім'я: Undefined Вік: 0
Ім'я: Tom Вік: 37
```

## Константи класу

Крім полів клас може визначати зберігання даних константи. На відміну від полів значення встановлюється один раз безпосередньо при їх оголошенні і згодом не може бути змінено. Крім того, константи зберігають деякі дані, які належать не до одного об'єкта, а до всього класу загалом. І для звернення до констант застосовується не ім'я об'єкта, а ім'я класу:

```csharp
Person tom = new Person();
tom.name = "Tom";
tom.age = 37;
tom.Print(); // Person: Tom - 37

Console.WriteLine(Person.type); // Person
// Person.type = "User"; // !Помилка: змінити константу не можна

class Person
{
    public const string type = "Person";
    public string name = "Undefined";
    public int age;
    public void Print() => Console.WriteLine($"{type}: {name} - {age}");
}
```

Тут у класі `Person` визначена константа `type`, яка зберігає назву класу:

```csharp
public const string type = "Person";
```

Назва класу не залежить від об'єкта. Ми можемо створити багато об'єктів `Person`, але назва класу від цього не повинна змінитися - вона стосується всіх об'єктів `Person` і не повинна змінюватися. Тому назву типу можна зберегти як константу.

Варто зазначити, що константі відразу при її визначенні необхідно надати значення.

Подібно до звичайних полів ми можемо звертатися до константів класу всередині цього класу. Наприклад, у методі `Print` значення константи виводиться на консоль.

Однак якщо ми хочемо звернутися до константи поза її класом, то для звернення необхідно використовувати ім'я класу:

```csharp
Console.WriteLine(Person.type); // Person
```

Таким чином, якщо необхідно зберігати дані, які відносяться до всього класу в цілому, можна використовувати константи.

Додавання класу до Visual Studio

Зазвичай класи розміщуються в окремі файли. Нерідко одному класу призначений один файл. І Visual Studio надає за замовчуванням вбудовані шаблони для додавання класу.

Для додавання класу натисніть у Visual Studio правою кнопкою миші назву проекту.

У контекстному меню, що з'явилося, виберемо пункт `Add -> New Item...` (або `Add -> Class...`)

У вікні додавання нового елемента, що відкрилося, переконаємося, що в центральній частині з шаблонами елементів у нас вибраний пункт `Class`.

![Рисунок з оригінального документа](assets/docx/image41.png)

А внизу вікна в полі `Name` введемо назву класу, що додається - нехай він буде називатися `Person`:

![Рисунок з оригінального документа](assets/docx/image42.png)

В якості назви класу можна вводити як `Person`, так і `Person.cs`. І після натискання на кнопку додавання до проекту буде додано новий клас

![Рисунок з оригінального документа](assets/docx/image43.png)

У файлі `Person.cs` визначимо наступний код:

```csharp
class Person
{
    public string name = "Undefined";
    public void Print()
    {
        Console.WriteLine($"Person {name}");
    }
}
```

Тут визначено клас `Person` з одним полем `name` та методом `Print`.

У файлі `Program.cs`, який представляє основний файл програми, використовуємо клас `Person`:

```csharp
Person tom = new Person();
tom.name = "Tom";
tom.Print(); // Person Tom
```

![Рисунок з оригінального документа](assets/docx/image44.png)

Таким чином, ми можемо визначати класи в окремих файлах та використовувати їх у програмі.
