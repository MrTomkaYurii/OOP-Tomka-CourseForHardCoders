## 3.1. Класи та об'єкти

### Проблема без класів

Уявімо що нам потрібно зберігати дані про кількох пацієнтів клініки. Без власних типів програма виглядатиме так:

```csharp
string patient1Name = "Олена Коваль";
string patient1Phone = "0501234567";
int patient1Age = 32;

string patient2Name = "Михайло Бондар";
string patient2Phone = "0671234567";
int patient2Age = 45;
```

Уже з двома пацієнтами — шість змінних. З десятьма — шістдесят. Їх легко переплутати, важко передавати між методами, неможливо зберегти у зручному масиві. І жодного захисту від помилки — ніщо не заважає записати вік у поле телефону.

**Клас вирішує цю проблему.** Він дозволяє визначити новий тип, який об'єднує пов'язані дані та дії над ними в одній структурі.

---

### Клас і об'єкт

Два ключових поняття ООП:

- **Клас** — це опис (шаблон, креслення), що визначає які поля та методи матиме кожен об'єкт цього типу.
- **Об'єкт** — конкретний екземпляр класу в пам'яті, з реальними значеннями полів.

Аналогія: клас — це бланк анкети пацієнта, однаковий для всіх. Об'єкт — заповнена анкета конкретної людини.

![Клас Patient та два об'єкти-екземпляри](_assets/03-classes/class-vs-object.svg)

З одного класу можна створити скільки завгодно незалежних об'єктів — кожен зберігає власні дані.

---

### Оголошення класу

Клас оголошується ключовим словом `class`:

```csharp
class Patient
{
    // поля і методи
}
```

Ім'я класу — з великої літери, стиль **PascalCase**. Тіло класу — у фігурних дужках.

---

### Поля

Поля — це змінні на рівні класу, що зберігають стан кожного об'єкта. На відміну від локальних змінних, поля доступні у будь-якому методі класу:

```csharp
class Patient
{
    public string Name = "Невідомий";
    public string Phone = "";
    public int Age;
}
```

Модифікатор `public` робить поле видимим ззовні класу. Якщо поле не ініціалізоване — воно отримує значення за замовчуванням: числові типи → `0`, `string` → `null`, `bool` → `false`.

> **Зверніть увагу:** прямий доступ до полів ззовні (`patient.Name = "..."`) — це найпростіший підхід для початку. У наступних розділах ми побачимо чому в реальних проєктах поля роблять `private` і використовують властивості.

---

### Методи

Методи визначають поведінку об'єкта. Метод класу має прямий доступ до всіх його полів:

```csharp
class Patient
{
    public string Name = "Невідомий";
    public string Phone = "";
    public int Age;

    public void Display()
    {
        Console.WriteLine($"Пацієнт: {Name}, тел: {Phone}, вік: {Age}");
    }

    public string GetShortInfo()
    {
        return $"{Name} ({Age} р.)";
    }
}
```

Метод `Display()` виводить повну інформацію. Метод `GetShortInfo()` повертає рядок — він може бути використаний у будь-якому виразі.

---

### Створення об'єкта

Об'єкт створюється оператором `new`:

```csharp
Patient patient1 = new Patient();
```

Що відбувається у цьому рядку:
1. `new` виділяє пам'ять для об'єкта
2. Викликається конструктор (у цьому випадку автоматичний, без параметрів)
3. Змінна `patient1` отримує **посилання** на створений об'єкт

З C# 9 тип можна не повторювати, якщо він вже вказаний у змінній:

```csharp
Patient patient1 = new(); // те саме що new Patient()
```

---

### Точкова нотація

Доступ до полів і методів об'єкта — через крапку:

```csharp run
class Patient
{
    public string Name = "Невідомий";
    public string Phone = "";
    public int Age;

    public void Display()
    {
        Console.WriteLine($"Пацієнт: {Name}, тел: {Phone}, вік: {Age}");
    }

    public string GetShortInfo()
    {
        return $"{Name} ({Age} р.)";
    }
}

// читаємо значення полів
Patient patient1 = new Patient();
Console.WriteLine(patient1.Name);  // Невідомий
Console.WriteLine(patient1.Age);   // 0

// встановлюємо нові значення
patient1.Name = "Олена Коваль";
patient1.Phone = "0501234567";
patient1.Age = 32;

// викликаємо методи
patient1.Display();
Console.WriteLine(patient1.GetShortInfo());
```

---

### Кілька об'єктів

Кожен об'єкт — **незалежний**. Зміна полів одного не впливає на інший:

```csharp
Patient patient1 = new Patient();
patient1.Name = "Олена Коваль";
patient1.Age = 32;

Patient patient2 = new Patient();
patient2.Name = "Михайло Бондар";
patient2.Age = 45;

patient1.Display(); // Пацієнт: Олена Коваль, тел: , вік: 32
patient2.Display(); // Пацієнт: Михайло Бондар, тел: , вік: 45

// змінюємо patient1 — patient2 не змінюється
patient1.Age = 33;
patient2.Display(); // вік: 45 — не змінився
```

---

### Масив об'єктів

Клас — це повноцінний тип, і об'єкти можна зберігати у масиві:

```csharp
Patient[] patients = new Patient[3];

patients[0] = new Patient();
patients[0].Name = "Олена Коваль";
patients[0].Age = 32;

patients[1] = new Patient();
patients[1].Name = "Михайло Бондар";
patients[1].Age = 45;

patients[2] = new Patient();
patients[2].Name = "Ірина Мельник";
patients[2].Age = 28;

// виводимо всіх пацієнтів
foreach (Patient p in patients)
{
    p.Display();
}
```

```
Пацієнт: Олена Коваль, тел: , вік: 32
Пацієнт: Михайло Бондар, тел: , вік: 45
Пацієнт: Ірина Мельник, тел: , вік: 28
```

Саме таку структуру — клас + масив об'єктів — ми будемо розвивати впродовж усього курсу.

---

### Константи класу

Константи зберігають значення, що належить **всьому класу**, а не конкретному об'єкту. Оголошуються з `const`, отримують значення лише при оголошенні і не можуть бути змінені:

```csharp
class Patient
{
    public const int MaxNameLength = 100;
    public const string DefaultPhone = "невідомий";

    public string Name = "Невідомий";
    public string Phone = DefaultPhone;  // використовуємо константу
    public int Age;

    public void Display()
    {
        Console.WriteLine($"Пацієнт: {Name}, тел: {Phone}, вік: {Age}");
    }
}
```

Звернення до константи — через ім'я **класу**, не об'єкта (бо вона не належить жодному конкретному об'єкту):

```csharp
Console.WriteLine(Patient.MaxNameLength); // 100
// Patient.MaxNameLength = 50;           // помилка: змінити константу неможливо
```

### Клас в окремому файлі

У реальних проєктах кожен клас розміщується у власному файлі. Ім'я файлу має збігатися з ім'ям класу: `Patient.cs`.

**Patient.cs**
```csharp
class Patient
{
    public string Name = "Невідомий";
    public string Phone = "";
    public int Age;

    public void Display()
    {
        Console.WriteLine($"Пацієнт: {Name}, тел: {Phone}, вік: {Age}");
    }

    public string GetShortInfo()
    {
        return $"{Name} ({Age} р.)";
    }
}
```

**Program.cs**
```csharp
Patient p = new Patient();
p.Name = "Олена Коваль";
p.Phone = "0501234567";
p.Age = 32;
p.Display();
```

Обидва файли належать одному проєкту — компілятор C# бачить усі класи автоматично, без жодного `import` чи `include`.

---

### Підсумок

| Поняття | Що це |
|---|---|
| `class` | Шаблон нового типу з полями і методами |
| Поле | Змінна, що зберігає стан об'єкта |
| Метод | Функція, що визначає поведінку об'єкта |
| `new` | Створює об'єкт у пам'яті, повертає посилання |
| `public` | Поле або метод доступні ззовні класу |
| `const` | Значення належить класу, не об'єкту; не змінюється |

У наступному параграфі ми розглянемо **конструктори** — спеціальні методи, що дозволяють задавати початкові значення полів одразу при створенні об'єкта, а не після.
