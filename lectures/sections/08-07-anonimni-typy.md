## 8.7. Анонімні типи

Анонімні типи дозволяють створити об'єкт із деяким набором властивостей без визначення класу. Анонімний тип визначається за допомогою ключового слова var та ініціалізатора об'єктів:

```csharp
var user = new { Name = "Tom", Age = 34 };
Console.WriteLine(user.Name);
```

В даному випадку user - це об'єкт анонімного типу, який має дві властивості Name і Age. І ми також можемо використовувати його властивості, як і звичайні об'єкти класів. Однак тут є обмеження - властивості анонімних типів доступні лише для читання.

При цьому під час компіляції компілятор сам створюватиме для нього ім'я типу та використовуватиме це ім'я при зверненні до об'єкта. Нерідко анонімні типи мають ім'я на кшталт "<>f__AnonymousType0'2".

Для середовища CLR анонімні типи будуть також, як і класи, представляти посилальний тип.

Якщо у програмі використовуються кілька об'єктів анонімних типів з однаковим набором властивостей, то для них компілятор створить одне визначення анонімного типу:

```csharp
var user = new { Name = "Tom", Age = 34 };
var student = new { Name = "Alice", Age = 21 };
var manager = new { Name = "Bob", Age = 26, Company = "Microsoft" };
Console.WriteLine(user.GetType().Name); // <>f__AnonymousType0'2
Console.WriteLine(student.GetType().Name); // <>f__AnonymousType0'2
Console.WriteLine(manager.GetType().Name); // <>f__AnonymousType1'3
```

Тут user і student будуть мати те саме визначення анонімного типу. Однак подібні об'єкти не можна перетворити до іншого типу, наприклад, класу, навіть якщо він має подібний набір властивостей.

Слід враховувати, що властивості анонімного об'єкта доступні для встановлення лише в ініціалізаторі. Поза ініціалізатором присвоїти їм значення ми не можемо. Тому, наприклад, у наступному випадку ми зіткнемося з помилкою:

```csharp
var student = new { Name = "Alice", Age = 21 };
student.Age = 32;   // ! Помилка
```

Крім використаної вище форми ініціалізації, коли ми присвоюємо властивостям деякі значення, також можна використовувати ініціалізатори з проекцією (projection initializers), коли ми можемо передати в ініціалізатор деякі ідентифікатори, імена яких будуть використовуватись як назви властивостей:

```csharp
Person tom = new Person("Tom");
int age = 34;
var student = new { tom.Name, age }; // ініціалізатор із проекцією
Console.WriteLine(student.Name);
Console.WriteLine(student.age);

class Person
{
    public string Name { get; set; }
    public Person(string name) => Name = name;
}
```

У разі визначення анонімного об'єкта фактично буде ідентично наступному:

```csharp
var student = new { Name = tom.Name, age = age };
```

Назви властивостей та змінних (Name та age) будуть використовуватися як назви властивостей об'єкта.

Також можна визначати масиви об'єктів анонімних типів:

```csharp
var people = new[]
{
    new { Name = "Tom" },
    new { Name = "Bob" }
};
foreach (var p in people)
{
    Console.WriteLine(p.Name);
}
```

Навіщо потрібні анонімні типи? Іноді виникає завдання використовувати один тип в одному вузькому контексті або навіть один раз. Створення класу для такого типу може бути надлишковим. Якщо нам захочеться додати властивість, ми відразу ж на місці анонімного об'єкта це можемо зробити. У випадку класу доведеться змінювати ще й клас, який може більше ніде не використовуватися. Типова ситуація - отримання результату вибірки з бази даних: об'єкти використовуються лише для отримання вибірки, часто більше ніде не використовуються, і класи їм створювати було зайве. А ось анонімний об'єкт чудово підходить для тимчасового зберігання вибірки.
