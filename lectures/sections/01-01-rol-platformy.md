## 1.1. Роль платформи

Коли говорять про C#, майже завжди поруч з'являється слово `.NET`. Це не випадково: C# є мовою програмування, а .NET є платформою, на якій ця мова найчастіше використовується. Мова описує синтаксис, типи, класи, методи, оператори та правила написання програм. Платформа дає середовище виконання, бібліотеки, інструменти збирання, систему пакетів і набір технологій для різних типів застосунків.

Тому C# і .NET не треба ототожнювати. C# - це мова. .NET - це екосистема, у якій ця мова розкривається найповніше.

Спрощено це можна уявити так:

```text
C# код студента
      |
      v
Компілятор C#
      |
      v
Проміжна мова CIL + метадані
      |
      v
Середовище виконання .NET
      |
      v
Машинний код для конкретного процесора
```

### Що дає .NET

.NET бере на себе багато роботи, яку в низькорівневих мовах програміст змушений контролювати самостійно. Платформа не просто запускає програму, а організовує її виконання: завантажує збірки, перевіряє типи, керує пам'яттю, виконує JIT-компіляцію, дає доступ до стандартних бібліотек і забезпечує взаємодію з операційною системою.

Для студента це означає просту річ: на початку курсу можна зосередитися на логіці програми, класах, методах, об'єктах і структурі коду, а не на ручному виділенні та звільненні пам'яті.

### Підтримка кількох мов

Основою виконання програм у .NET є CLR - Common Language Runtime, тобто загальномовне середовище виконання. Воно не прив'язане лише до C#. На платформі .NET також використовують F#, Visual Basic та інші мови, які можуть компілюватися у спільне проміжне подання.

Коли код C#, F# або іншої .NET-мови компілюється, результатом є не одразу машинні інструкції конкретного процесора, а проміжна мова CIL (Common Intermediate Language) разом із метаданими. Завдяки цьому різні мови можуть працювати в межах однієї платформи, використовувати одні й ті самі бібліотеки та навіть взаємодіяти в межах одного рішення.

Наприклад, теоретично одна частина великої системи може бути написана на C#, інша - на F#, а використовувати вони можуть спільні типи й бібліотеки .NET.

### Кросплатформність

Сучасний .NET є кросплатформовим. Це означає, що застосунки на C# можна створювати не лише для Windows, а й для Linux, macOS, хмарних середовищ, контейнерів, мобільних і веб-сценаріїв. Звісно, не кожна конкретна технологія працює всюди: наприклад, WPF і Windows Forms орієнтовані на Windows, тоді як ASP.NET Core, консольні застосунки, бібліотеки класів і багато серверних рішень можуть працювати на різних операційних системах.

Для навчального курсу це важливо з практичної причини: студент може писати C#-код у Windows, Linux або macOS, використовувати .NET CLI, запускати консольні програми, виконувати тести й працювати з Git без прив'язки до однієї IDE.

### Бібліотека класів

Одна з найсильніших сторін .NET - велика стандартна бібліотека класів. У ній уже є готові типи для роботи з рядками, датами, колекціями, файлами, потоками, мережею, JSON, базами даних, асинхронністю та багатьма іншими задачами.

Коли ми пишемо:

```csharp
Console.WriteLine("Hello, World!");
```

ми вже використовуємо бібліотеку .NET. Тип `Console` не є частиною синтаксису мови C#; це клас із бібліотеки платформи, який дає зручний доступ до консолі.

Саме тому в курсі важливо поступово розрізняти три рівні:

- синтаксис C#;
- типи та можливості стандартної бібліотеки .NET;
- архітектуру власної програми.

### Різноманітність технологій

На .NET можна створювати різні типи програм:

- консольні застосунки;
- бібліотеки класів;
- веб API та вебзастосунки на ASP.NET Core;
- настільні застосунки для Windows на WPF, Windows Forms або WinUI;
- кросплатформові застосунки з .NET MAUI;
- інтерактивні вебінтерфейси з Blazor;
- фонові служби, мікросервіси, хмарні сервіси та інструменти командного рядка;
- ігрові або прикладні системи, якщо використовується відповідний рушій чи бібліотека.

У цьому курсі ми починаємо з консольних застосунків, бо вони дозволяють не відволікатися на інтерфейс і зосередитися на головному: типах, об'єктах, класах, методах, інкапсуляції, наслідуванні та поліморфізмі.

### Продуктивність і розвиток платформи

Сучасний .NET активно розвивається. Microsoft щороку випускає нові основні версії платформи, а разом із ними оновлюються компілятор, runtime, бібліотеки, ASP.NET Core, Entity Framework Core та інші частини екосистеми. Для реальних проєктів це означає постійне покращення продуктивності, інструментів і підтримки сучасних сценаріїв розробки.

Продуктивність .NET забезпечується не однією магічною можливістю, а сукупністю рішень: JIT-компіляцією, оптимізаціями runtime, ефективною роботою з пам'яттю, сучасними колекціями, асинхронними API та можливістю за потреби використовувати AOT-компіляцію в окремих сценаріях.

### Автоматичне керування пам'яттю

Важлива особливість .NET - автоматичне збирання сміття (garbage collection). Коли програма створює об'єкти, вони займають пам'ять. Якщо об'єкти більше не використовуються, CLR може звільнити цю пам'ять автоматично.

Це не означає, що про ресурси взагалі не треба думати. Файли, мережеві підключення, підключення до баз даних та інші зовнішні ресурси все одно потрібно закривати коректно. Але більшість повсякденної роботи з пам'яттю для звичайних об'єктів бере на себе платформа.

Для порівняння: у C++ програміст часто сам відповідає за те, коли виділити пам'ять і коли її звільнити. У C# типовий навчальний код виглядає простіше, бо CLR контролює життєвий цикл керованих об'єктів.

### Що важливо запам'ятати

C# - це мова програмування, а .NET - платформа для створення та виконання програм. CLR виконує керований код, стандартна бібліотека дає готові класи для типових задач, а вся екосистема .NET дозволяє будувати консольні, веб, настільні, мобільні та серверні застосунки.

Для цього курсу .NET є робочим середовищем, у якому ми поступово переходимо від простих інструкцій до повноцінного об'єктно-орієнтованого проєктування.
