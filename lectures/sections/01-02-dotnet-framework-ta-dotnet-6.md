## 1.2. .NET Framework та сучасний .NET

Історично .NET розпочинався як платформа для Windows під назвою .NET Framework. Вона довго була основною платформою для розробки Windows-застосунків, корпоративних систем, вебзастосунків на ASP.NET і настільних програм на Windows Forms або WPF.

Однак .NET Framework і сучасний .NET - це не одне й те саме.

### .NET Framework

.NET Framework - це класична Windows-орієнтована платформа. Вона тісно пов'язана з операційною системою Windows і досі використовується в багатьох старих корпоративних системах. Якщо в організації є велика програма, написана багато років тому на Windows Forms, WPF або старому ASP.NET, вона цілком може працювати саме на .NET Framework.

Актуальна гілка .NET Framework залишається підтримуваною для сумісності, але це вже не основний напрям розвитку платформи. Нові можливості C#, runtime, продуктивність, хмарні сценарії та кросплатформова розробка розвиваються насамперед у сучасному .NET.

Важливо: .NET Framework є Windows-only технологією. Якщо проєкт має працювати однаково на Windows, Linux і macOS, для нового коду зазвичай обирають не .NET Framework, а сучасний .NET.

### .NET Core і перехід до сучасного .NET

Щоб зробити платформу відкритішою, швидшою і кросплатформовою, Microsoft почала розвивати .NET Core. Це була нова гілка .NET, яка могла працювати на різних операційних системах і краще підходила для сучасних веб, серверних і хмарних застосунків.

Після .NET Core 1, 2 і 3 Microsoft перейшла до простішої назви: `.NET`. Починаючи з .NET 5, слово `Core` з назви прибрали. Тому коли сьогодні говорять просто `.NET`, зазвичай мають на увазі сучасну кросплатформову платформу, а не старий .NET Framework.

Умовно розвиток можна подати так:

```text
.NET Framework
      |
      |  класична Windows-платформа
      v
.NET Core
      |
      |  кросплатформовий перезапуск
      v
.NET 5, .NET 6, .NET 7, .NET 8, .NET 9, .NET 10 ...
```

### Актуальні версії

Станом на травень 2026 року актуальною LTS-версією є .NET 10. Також підтримуються .NET 8 LTS і .NET 9 STS. .NET 6, який раніше часто використовували в навчальних матеріалах як актуальну LTS-версію, уже не є поточною підтримуваною гілкою.

У .NET є два основні типи релізів:

- LTS (Long Term Support) - версія з довшим строком підтримки, яку зручно брати для стабільних навчальних, корпоративних і production-проєктів;
- STS (Standard Term Support) - версія з коротшим строком підтримки, яка швидше приносить нові можливості.

Для навчального курсу найкраща стратегія проста: використовувати актуальну LTS-версію .NET, якщо викладач або проєкт не вимагає іншого. Це дає стабільність, нормальну підтримку інструментів і менше несподіванок під час встановлення SDK.

### Що встановлює розробник

Для розробки на C# потрібен не лише runtime, а SDK.

Runtime дозволяє запускати вже зібрані програми. SDK містить усе необхідне для розробки: компілятор, інструменти командного рядка, шаблони проєктів, засоби збирання та сам runtime.

Перевірити встановлену версію SDK можна командою:

```bash
dotnet --version
```

Переглянути всі встановлені SDK:

```bash
dotnet --list-sdks
```

Створити простий консольний проєкт:

```bash
dotnet new console -n HelloCourse
cd HelloCourse
dotnet run
```

Ці команди важливі, бо вони показують, що .NET - це не тільки Visual Studio. Працювати з C# можна і через командний рядок, використовуючи .NET CLI.

### Що обирати для нових проєктів

Для нових навчальних і більшості нових реальних проєктів варто орієнтуватися на сучасний .NET. .NET Framework має сенс тоді, коли потрібно підтримувати стару Windows-систему або використовувати технологію, яка прив'язана саме до .NET Framework.

У цьому курсі під `.NET` мається на увазі сучасна кросплатформова платформа. Якщо в тексті згадується .NET Framework, то це робиться для історичного контексту або для пояснення різниці між старою та сучасною гілками розвитку.

### Що важливо запам'ятати

.NET Framework - це класична Windows-платформа, яка важлива для підтримки старих систем. Сучасний .NET - це кросплатформова платформа для нової розробки. .NET 6 уже не треба сприймати як поточну версію для нового курсу; для актуального навчання слід орієнтуватися на підтримувані версії, насамперед на поточну LTS-гілку.
