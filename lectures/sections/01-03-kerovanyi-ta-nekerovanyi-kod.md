## 1.3. Керований та некерований код

Програму, написану на C# для .NET, часто називають керованим кодом (managed code). Це означає, що її виконання відбувається під контролем CLR - Common Language Runtime.

CLR не просто запускає програму. Вона завантажує збірки, працює з метаданими, перевіряє типи, виконує JIT-компіляцію, керує пам'яттю, обробляє винятки та надає інші служби виконання.

### Керований код

Керований код - це код, який компілюється у проміжну мову CIL і виконується середовищем .NET. До керованих мов належать C#, F#, Visual Basic та інші мови, які можуть працювати поверх CLR.

Коли ми пишемо такий код:

```csharp
string name = "Ірина";
Console.WriteLine($"Привіт, {name}!");
```

ми не керуємо вручну тим, де саме в пам'яті зберігається рядок, коли його треба звільнити і як саме програма буде перетворена на машинні інструкції. Значну частину цієї роботи бере на себе runtime.

Керований код отримує від CLR кілька важливих переваг:

- автоматичне керування пам'яттю для об'єктів;
- перевірку типів і безпечніші правила роботи з даними;
- єдину модель винятків;
- доступ до стандартної бібліотеки .NET;
- можливість взаємодії між різними .NET-мовами;
- підтримку діагностики, профілювання та налагодження.

### Некерований код

Некерований код (unmanaged code) - це код, виконання якого не контролюється CLR. Типовий приклад - програми або бібліотеки, написані на C чи C++, які компілюються безпосередньо у машинний код для конкретної платформи.

У такому випадку операційна система завантажує програму, а відповідальність за багато деталей лягає на програміста або на використані бібліотеки. Зокрема, у низькорівневому коді частіше доводиться явно думати про виділення і звільнення пам'яті, роботу з вказівниками, коректне закриття ресурсів і безпеку доступу до пам'яті.

Це не означає, що некерований код "поганий". Він потрібен для системного програмування, драйверів, ігрових рушіїв, високопродуктивних бібліотек, роботи з обладнанням та багатьох інших задач. Але такий код вимагає більше дисципліни й уважності.

### Порівняння

```text
Керований код (.NET)
  - виконується під контролем CLR;
  - компілюється у CIL;
  - отримує garbage collection;
  - має метадані про типи;
  - зручний для прикладної розробки.

Некерований код
  - виконується без CLR;
  - часто компілюється одразу в машинний код;
  - може вимагати ручного керування пам'яттю;
  - ближчий до операційної системи й апаратури;
  - корисний у системних і високопродуктивних сценаріях.
```

### Взаємодія з некерованим кодом

.NET не ізольований від зовнішнього світу. Керований код може взаємодіяти з некерованими бібліотеками. Наприклад, C#-програма може викликати функції з Windows API або з нативної бібліотеки, написаної на C чи C++.

Таку взаємодію називають interop - interoperability. У C# для цього існують спеціальні механізми, наприклад P/Invoke. Вони потрібні не на початку курсу, але важливо розуміти саму ідею: .NET дозволяє залишатися у зручному керованому середовищі, але за потреби звертатися до нативного коду.

При цьому межа між керованим і некерованим світом є відповідальною зоною. Якщо керований код передає дані в некеровану бібліотеку, треба правильно описати типи, пам'ять і правила виклику. Інакше можна отримати помилки, які складніше діагностувати, ніж звичайні винятки C#.

### Unsafe-код у C#

C# також дозволяє писати небезпечний код (unsafe code), у якому можна працювати з вказівниками. Такий код усе ще може бути частиною .NET-програми, але окремі його ділянки виходять за межі звичайної безпечної моделі C#.

На початку навчання unsafe-код не потрібен. Навпаки, краще спершу добре засвоїти типи, об'єкти, масиви, методи, класи та колекції. До низькорівневих можливостей варто переходити лише тоді, коли зрозуміло, яку конкретну проблему вони розв'язують.

### Що важливо запам'ятати

Керований код виконується під контролем CLR і отримує служби платформи .NET: JIT-компіляцію, керування пам'яттю, перевірку типів і стандартну бібліотеку. Некерований код виконується без такого контролю і дає більше низькорівневої свободи, але вимагає більшої відповідальності.

Для цього курсу основним буде керований код C#, бо саме він є природним середовищем для вивчення об'єктно-орієнтованого програмування.
