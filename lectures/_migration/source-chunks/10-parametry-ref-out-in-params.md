# 2.19. Передача параметрів за посиланням та значенням. Вихідні параметри

Існує два способи передачі параметрів методу у мові C#: за значенням і за посиланням.

## Передача параметрів за значенням

Найбільш простий спосіб передачі параметрів представляє передача за значенням, по суті, це звичайний спосіб передачі параметрів:

```csharp
void Increment(int n)
{
    n++;
    Console.WriteLine($"Кількість у методі Increment: {n}");
}

int number = 5;
Console.WriteLine($"Число до методу Increment: {number}");
Increment(number);
Console.WriteLine($"Число після методу Increment: {number}");
```

Консольний вивід:

![Рисунок з оригінального документа](_assets/_docx/image37.png)

При передачі аргументів параметрам за значенням параметр методу отримує не саму змінну, а її копію і далі працює із цією копією незалежно від самої змінної.

Так, вище за виклик метод `Increment` отримує копію змінної `number` і збільшує значення цієї копії. Тому в самому методі `Increment` бачимо, що значення параметра `n` збільшилося на 1, але після виконання методу змінна `number` має колишнє значення - 5. Тобто змінюється копія, а сама змінна не змінюється.

## Передача параметрів за посиланням та модифікатор ref

При надсиланні параметрів за посиланням перед параметрами використовується модифікатор `ref`:

```csharp
void Increment(ref int n)
{
    n++;
    Console.WriteLine($"Кількість у методі Increment: {n}");
}

int number = 5;
Console.WriteLine($"Число до методу Increment: {number}");
Increment(ref number);
Console.WriteLine($"Число після методу Increment: {number}");
```

Консольний вивід:

![Рисунок з оригінального документа](_assets/_docx/image38.png)

При передачі значень параметрів за посиланням метод отримує адресу змінної в пам'яті. І, таким чином, якщо в методі змінюється значення параметра, що передається за посиланням, то також змінюється значення змінної, яка передається на його місце.

Так, у метод `Increment` передається посилання на саму змінну `number` у пам'яті. І якщо значення параметра `n` в `Increment` змінюється, то це призводить і до зміни змінної `number`, так як і параметр `n` і змінна `number` вказують на ту саму адресу в пам'яті.

Зверніть увагу, що модифікатор `ref` вказується як перед параметром під час оголошення методу, так і під час виклику методу перед аргументом, який передається параметру.

## Вихідні параметри. Модифікатор out

Вище ми використали вхідні параметри. Але параметри можуть бути вихідними. Щоб зробити параметр вихідним, перед ним ставиться модифікатор `out`:

```csharp
void Sum(int x, int y, out int result)
{
    result = x + y;
}
```

Тут результат повертається не через оператор `return`, а через вихідний параметр `result`. Використання у програмі:

```csharp
void Sum(int x, int y, out int result)
{
    result = x + y;
}

int number;

Sum(10, 15, out number);

Console.WriteLine(number); // 25
```

Причому, як і у випадку з `ref`, ключове слово `out` використовується як при визначенні методу, так і при його виклику.

Також зверніть увагу, що методи, які використовують такі параметри, обов'язково повинні надавати їм певне значення. Тобто наступний код буде неприпустимим, тому що в ньому для `out`-параметра не вказано жодного значення:

```csharp
void Sum(int x, int y, out int result)
{
    Console.WriteLine(x + y);
}
```

Принадність використання подібних параметрів полягає в тому, що ми можемо повернути з методу не одне значення, а кілька. Наприклад:

```csharp
void GetRectangleData(int width, int height, out int rectArea, out int rectPerimetr)
{
    rectArea = width * height;
    // площа прямокутника - добуток ширини на висоту
    rectPerimetr = (width + height) * 2;
    // периметр прямокутника - сума довжин всіх сторін
}

int area;
int perimetr;

GetRectangleData(10, 20, out area, out perimetr);

Console.WriteLine($"Площа прямокутника: {area}"); // 200
Console.WriteLine($"Периметр прямокутника: {perimetr}"); // 60
```

Тут ми маємо метод `GetRectangleData`, який отримує ширину і висоту прямокутника (параметри `width` і `height`). А два вихідні параметри ми використовуємо для підрахунку площі та периметра прямокутника.

При цьому можна визначати змінні, які передаються `out`-параметрам безпосередньо при виклику методу. Тобто ми можемо скоротити попередній приклад так:

```csharp
void GetRectangleData(int width, int height, out int rectArea, out int rectPerimetr)
{
    rectArea = width * height;
    rectPerimetr = (width + height) * 2;
}

GetRectangleData(10, 20, out int area, out int perimetr);

Console.WriteLine($"Площа прямокутника: {area}"); // 200
Console.WriteLine($"Периметр прямокутника: {perimetr}"); // 60
```

При цьому, якщо нам невідомий тип значень, які будуть присвоєні параметрам, ми можемо для їх визначення використовувати оператор `var`:

```csharp
GetRectangleData(10, 20, out var area, out var perimetr);

Console.WriteLine($"Площа прямокутника: {area}"); // 200
Console.WriteLine($"Периметр прямокутника: {perimetr}"); // 60
```

## Вхідні параметри. Модифікатор in

Крім вихідних параметрів з модифікатором `out`, метод може використовувати вхідні параметри з модифікатором `in`. Модифікатор `in` вказує, що цей параметр передаватиметься у метод за посиланням, проте всередині методу значення параметра не можна буде змінити. Наприклад, візьмемо наступний метод:

```csharp
void GetRectangleData(in int width, in int height, out int rectArea, out int rectPerimetr)
{
    // width = 25; // не можна змінити, оскільки width - вхідний параметр
    rectArea = width * height;
    rectPerimetr = (width + height) * 2;
}

int w = 10;
int h = 20;
GetRectangleData(w, h, out var area, out var perimetr);

Console.WriteLine($"Площа прямокутника: {area}"); // 200
Console.WriteLine($"Периметр прямокутника: {perimetr}"); // 60
```

В даному випадку через вхідні параметри `width` і `height` методу передаються значення, але в самому методі ми не можемо змінити значення цих параметрів, так як вони визначені з модифікатором `in`.

Передача за посиланням у деяких випадках може збільшити продуктивність, а використання оператора `in` вказує, що передані параметри не можна змінити в цьому методі.

# 2.20. Масив параметрів та ключове слово params

В попередніх прикладах ми використовували постійне число параметрів. Але, використовуючи ключове слово `params`, ми можемо передавати невизначену кількість параметрів:

```csharp
void Sum(params int[] numbers)
{
    int result = 0;
    foreach (var n in numbers)
    {
        result += n;
    }
    Console.WriteLine(result);
}

int[] nums = { 1, 2, 3, 4, 5 };
Sum(nums);
Sum(1, 2, 3, 4);
Sum(1, 2, 3);
Sum();
```

Сам параметр з ключовим словом `params` щодо методу повинен представляти одномірний масив того типу, дані якого ми збираємося використовувати. При виклику методу на місце параметра з модифікатором `params` ми можемо передати як окремі значення, так і масив значень, або взагалі не передавати параметри. Кількість переданих значень методу невизначена, проте всі ці значення повинні відповідати типу параметра з `params`.

Якщо нам треба передати якісь інші параметри, то вони повинні вказуватися до параметра з ключовим словом `params`:

```csharp
void Sum(int initialValue, params int[] numbers)
{
    int result = initialValue;
    foreach (var n in numbers)
    {
        result += n;
    }
    Console.WriteLine(result);
}

int[] nums = { 1, 2, 3, 4, 5 };
Sum(10, nums);
// число 10 - передається параметру initialValue
Sum(1, 2, 3, 4);
Sum(1, 2, 3);
Sum(20);
```

Тут метод `Sum` має обов'язковий параметр `initialValue`, тому при виклику методу йому потрібно обов'язково передати значення. Тому перше значення під час виклику методу передаватиметься цьому параметру.

Однак, після параметра з модифікатором `params` ми не можемо вказувати інші параметри. Тобто таке визначення методу неприпустимо:

```csharp
// Так НЕ працює
void Sum(params int[] numbers, int initialValue)
{
}
```

## Масив як параметр

Також цей спосіб передачі параметрів треба відрізняти від передачі масиву як параметр:

```csharp
void Sum(int[] numbers, int initialValue)
{
    int result = initialValue;
    foreach (var n in numbers)
    {
        result += n;
    }
    Console.WriteLine(result);
}

int[] nums = { 1, 2, 3, 4, 5 };
Sum(nums, 10);

// Sum(1, 2, 3, 4);
// так не можна - нам треба передати масив
```

Так як метод `Sum` приймає як параметр масив без ключового слова `params`, то при його виклику нам обов'язково треба передати як перший параметр масив. Крім того, на відміну від методу з параметром `params`, після параметра масиву можуть розташовуватися інші параметри.
