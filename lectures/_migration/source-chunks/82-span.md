
## 13.5. Span

Тип `Span` представляє безперервну область пам'яті. Мета цього типу - підвищити продуктивність та ефективність використання пам'яті. `Span` дозволяє уникнути додаткових виділень пам'яті під час операції з наборами даних. Оскільки `Span` є структурою, то об'єкт цього типу розташовується в стеку, а не в хіпі.

### Створення Span

Для створення об'єкта `Span` можна застосовувати один із його конструкторів:

- `Span()`: створює порожній об'єкт `Span`
- `Span(T[] array)`: створює об'єкт `Span` з масиву `array`
- `Span(void* pointer, int length)`: створює об'єкт `Span`, який отримує `length` байт пам'яті, починаючи з покажчика `pointer`
- `Span(T[] array, int start, int length)`: створює об'єкт `Span`, який отримує з масиву `array` `length` елементів, починаючи з індексу `start`

Наприклад, найпростіше створення `Span`:

```csharp
string[] people = new string[] { "Tom", "Alice", "Bob" };
Span<string> peopleSpan = new Span<string>(people);
```

Ми також можемо безпосередньо присвоїти масив, і він неявно буде перетворений на `Span`:

```csharp
string[] people = new string[] { "Tom", "Alice", "Bob" };
Span<string> peopleSpan = people;
```

Далі ми можемо отримувати, встановлювати або перебирати дані так само, як у масиві:

```csharp
string[] people = new string[] { "Tom", "Alice", "Bob" };
Span<string> peopleSpan = people;
peopleSpan[1] = "Ann"; // Перевстановлення значення елемента
Console.WriteLine(peopleSpan[2]); // Отримання елемента
Console.WriteLine(peopleSpan.Length); // Отримання довжини Span

// Перебір Span
foreach (var s in peopleSpan)
{
    Console.WriteLine(s);
}
```

Якщо `Span` поводиться зовні як масив, то в чому його перевага чи коли він нам може стати в нагоді? Розглянемо найпростішу ситуацію - у нас є масив зі значеннями денних температур повітря за місяць, і нам треба отримати їх два набори - набір температур за першу декаду і за останню декаду. Використовуючи масиви, ми могли б зробити так:

```csharp
int[] temperatures = new int[]
{
    10, 12, 13, 14, 15, 11, 13, 15, 16, 17,
    18, 16, 15, 16, 17, 14, 9, 8, 10, 11,
    12, 14, 15, 15, 16, 15, 13, 12, 12, 11
};
int[] firstDecade = new int[10];
// Виділяємо пам'ять для першої декади.
int[] lastDecade = new int[10];
// Виділяємо пам'ять для другої декади.
Array.Copy(temperatures, 0, firstDecade, 0, 10);
// Копіюємо дані в перший масив.
Array.Copy(temperatures, 20, lastDecade, 0, 10);
// Копіюємо дані в другий масив.
```

Для зберігання даних створюються два додаткові масиви для денних температур кожної декади. За допомогою методу `Array.Copy` дані з вихідного масиву `temperatures` копіюються в два інших масиви. Але суть у цьому випадку в тому, що для обох масивів ми змушені виділити пам'ять. Тобто обидва масиви по суті містять самі дані, що й `temperatures`, проте у окремих частинах пам'яті.

`Span` дозволяє працювати з пам'яттю ефективніше і уникнути непотрібних виділень пам'яті. Так, використовуємо замість масивів `Span`:

```csharp
int[] temperatures = new int[]
{
    10, 12, 13, 14, 15, 11, 13, 15, 16, 17,
    18, 16, 15, 16, 17, 14, 9, 8, 10, 11,
    12, 14, 15, 15, 16, 15, 13, 12, 12, 11
};
Span<int> temperaturesSpan = temperatures;

Span<int> firstDecade = temperaturesSpan.Slice(0, 10);
// Немає виділення пам'яті під дані.
Span<int> lastDecade = temperaturesSpan.Slice(20, 10);
// Немає виділення пам'яті під дані.
```

Для створення похідних об'єктів `Span` застосовується метод `Slice`, який із `Span` виділяє частину та повертає її у вигляді іншого об'єкта `Span`. Тепер об'єкти `Span` `firstDecade` і `lastDecade` працюють із тими самими даними, як і `temperaturesSpan`, а додатково пам'ять не виділяється. Тобто у всіх трьох випадках ми працюємо з тим самим масивом `temperatures`. Ми навіть можемо в одному `Span` змінити дані і дані зміняться в іншому:

```csharp
int[] temperatures = new int[]
{
    10, 12, 13, 14, 15, 11, 13, 15, 16, 17,
    18, 16, 15, 16, 17, 14, 9, 8, 10, 11,
    12, 14, 15, 15, 16, 15, 13, 12, 12, 11
};
Span<int> temperaturesSpan = temperatures;

Span<int> firstDecade = temperaturesSpan.Slice(0, 10);

temperaturesSpan[0] = 25; // Змінюємо в temperaturesSpan
Console.WriteLine(firstDecade[0]); // 25
```

За рахунок чого це досягається? Для розуміння роботи `Span` можна звернутися до коду типу. Зокрема, ми можемо в ньому побачити таку властивість:

```csharp
public readonly ref struct Span<T>
{
    // ...
    public ref T this[int index] { get { ... } }
    // ...
}
```

Тут ми бачимо, що індексатор повертає `ref`-посилання, завдяки чому ми отримуємо доступ безпосередньо до об'єкта та можемо його змінювати.

У цьому разі, звісно, переваги від відсутності виділення додаткової пам'яті під зберігання об'єктів мінімальні. Але при більш інтенсивній роботі з даними виграш у продуктивності неминуче має зростати.

### Методи Span

Основні методи `Span`:

- `void Fill(T value)`: заповнює всі елементи `Span` значенням `value`
- `T[] ToArray()`: перетворює `Span` на масив
- `Span<T> Slice(int start, int length)`: виділяє із `Span` `length` елементів починаючи з індексу `start` у вигляді іншого `Span`
- `void Clear()`: очищає `Span`
- `void CopyTo(Span<T> destination)`: копіює елементи поточного `Span` в інший `Span`
- `bool TryCopyTo(Span<T> destination)`: копіює елементи поточного `Span` в інший `Span`, але також повертає значення `bool`, яке вказує, чи вдало пройшла операція копіювання

### ReadOnlySpan

Структура `ReadOnlySpan` аналогічна `Span`, тільки призначена для незмінних даних. Наприклад:

```csharp
string text = "hello, world";
string worldString = text.Substring(startIndex: 7, length: 5); // Є виділення пам'яті під символи
ReadOnlySpan<char> worldSpan = text.AsSpan().Slice(start: 7, length: 5); // Немає виділення пам'яті під символи
// worldSpan[0] = 'a'; // Не можна змінити
Console.WriteLine(worldSpan[0]); // Виводимо перший символ

// Перебір символів
foreach (var c in worldSpan)
{
    Console.Write(c);
}
```

У даному випадку за допомогою методу `AsSpan()` перетворюємо рядок на об'єкт `ReadOnlySpan<char>` і потім виділяємо з нього діапазон символів `"world"`. Оскільки `ReadOnlySpan` призначений тільки для читання, то ми не можемо змінити через нього дані, але отримати можемо. В іншому робота з `ReadOnlySpan` йде так само, як із `Span`.

### Обмеження Span

Як структура, визначена з модифікатором `ref`, `Span` має ряд обмежень: вона не може бути присвоєна змінній типу `Object`, `dynamic` чи змінній типу інтерфейсу. Вона не може бути полем в об'єкті типу посилання, а тільки всередині `ref`-структур. Вона не може використовуватися в межах операцій `await` або `yield`.
