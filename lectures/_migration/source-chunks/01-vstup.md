
$11.4. JIT-компіляція

Код C# не виконується процесором безпосередньо в тому вигляді, у якому його написав програміст. Спочатку файл із розширенням `.cs` обробляє компілятор C#. Результатом є збірка - зазвичай файл `.dll` або `.exe`, усередині якого містяться CIL-інструкції та метадані.

CIL (Common Intermediate Language) - це проміжна мова платформи .NET. Вона не є мовою, якою зазвичай пише програміст, але саме в неї компілюється C#-код перед виконанням.

### Загальна схема виконання

```text
Program.cs
   |
   |  компілятор C#
   v
Збірка .dll або .exe
   |
   |  CIL + метадані
   v
CLR завантажує програму
   |
   |  JIT-компілятор
   v
Машинний код для поточного процесора
   |
   v
Виконання програми
```

JIT означає Just-In-Time - "саме вчасно". Ідея полягає в тому, що CIL перетворюється на машинний код не під час написання програми і не обов'язково під час її збирання, а під час виконання, коли конкретний метод дійсно знадобився.

### Як працює JIT

Коли програма запускається, CLR не обов'язково компілює весь код одразу. Якщо у програмі є багато класів і методів, але частина з них під час конкретного запуску не використовується, немає сенсу витрачати час на їх компіляцію.

Зазвичай процес виглядає так:

1. Програма запускається.
2. CLR завантажує потрібну збірку.
3. Коли певний метод викликається вперше, JIT-компілятор перетворює його CIL-код на машинний код.
4. Скомпільований машинний код виконується.
5. Якщо той самий метод викликається повторно, використовується вже підготовлений машинний код.

Тобто JIT працює поступово, за потреби. Це дозволяє не компілювати зайве і водночас адаптувати машинний код до конкретної архітектури процесора.

### Приклад на рівні ідеї

Нехай у програмі є два методи:

```csharp
static void Main()
{
    SayHello();
}

static void SayHello()
{
    Console.WriteLine("Привіт!");
}

static void UnusedMethod()
{
    Console.WriteLine("Цей метод може не викликатися під час запуску");
}
```

Під час конкретного запуску викликається `Main`, а з нього - `SayHello`. Метод `UnusedMethod` існує у збірці, але якщо до нього не буде звернення, runtime може взагалі не компілювати його в машинний код у цьому запуску.

Це і є практичний сенс Just-In-Time: компілювати те, що справді потрібно для виконання.

### CIL і метадані

Окрім CIL, збірка містить метадані. Метадані описують типи, методи, поля, властивості, параметри та іншу інформацію про код. Завдяки метаданим runtime може розуміти структуру програми, перевіряти типи, завантажувати класи та забезпечувати роботу рефлексії.

Тому .NET-збірка є не просто набором інструкцій. Вона містить самодостатній опис коду, з яким може працювати CLR.

### Чому це зручно

JIT-компіляція дає кілька переваг:

- одна і та сама CIL-збірка може виконуватися на різних платформах, якщо там є відповідний .NET runtime;
- машинний код генерується під конкретну архітектуру процесора;
- компілюється лише той код, який реально використовується;
- runtime може застосовувати оптимізації під час виконання;
- розробник працює з високорівневою мовою C#, а деталі перетворення бере на себе платформа.

Саме тому C# поєднує зручність високорівневої мови з досить високою продуктивністю виконання.

### JIT і AOT

JIT - не єдиний спосіб отримати машинний код у .NET. У сучасному .NET також існують сценарії AOT-компіляції (Ahead-Of-Time), коли код компілюється наперед, ще до запуску програми. AOT може бути корисним для швидшого старту, окремих хмарних сценаріїв, мобільних застосунків або середовищ, де небажано генерувати код під час виконання.

Однак для більшості навчальних консольних програм і звичайних серверних застосунків базове розуміння JIT залишається ключовим. Саме через JIT студентам найпростіше пояснити, як C#-код проходить шлях від `.cs` файлу до машинних інструкцій.

### Що важливо запам'ятати

C#-код спочатку компілюється у CIL і зберігається у збірці разом із метаданими. Під час запуску CLR завантажує збірку, а JIT-компілятор перетворює потрібні частини CIL на машинний код. Завдяки цьому .NET може поєднувати кросплатформовість, безпечне кероване виконання і високу продуктивність.
